### 2.2 섹터 카드 헤더

각 카드 상단에 `quant_sector_fire` 데이터 표시:

```
┌─ 🔥 AI반도체 ── FIRE B등급 55점 ─────────────────────────┐
│  외인5d: -6,559억  기관5d: +10,778억  연기금: +1,109억    │
│  ETF: KODEX 반도체 (091160)                              │
├──────────────────────────────────────────────────────────┤
│  (종목 테이블)                                            │
└──────────────────────────────────────────────────────────┘
```

**FIRE 등급별 카드 테두리 색상**:

| 등급 | 점수 | 테두리 색상 | 배경 |
|------|------|-----------|------|
| **S** | 80+ | `#FF4444` (빨강) | `rgba(255,68,68,0.05)` |
| **A** | 60~79 | `#FF6B35` (주황) | `rgba(255,107,53,0.05)` |
| **B** | 40~59 | `#FFB347` (노랑) | `rgba(255,179,71,0.03)` |
| **C** | 20~39 | `#999999` (회색) | 없음 |
| **D** | <20 | 표시 안 함 | — |

### 2.3 종목 테이블 (카드 내부)

각 카드 안의 종목 테이블 — **현재 컬럼 유지하되 간소화**:

| 컬럼 | 소스 필드 | 표시 |
|------|----------|------|
| 종목 | `name` + `ticker` | 종목명 (티커는 작은 글씨) |
| 종가 | `close` | 원 단위 |
| 등락 | `chg_1d` | 양수=빨강, 음수=파랑 |
| 점수 | `buy_score` | 숫자 |
| 등급 | `buy_grade` | 배지 (강력=빨강, BUY=주황, WATCH=회색) |
| MA20 | `ma20_dev` | +N.N% 형식 |
| RSI | `rsi` | 숫자 |
| 외5d | `fgn_5d` | 억원, 색상 (양=빨강, 음=파랑) |
| 기5d | `inst_5d` | 억원, 색상 |
| 근거 | `buy_reasons` | 첫 번째 사유만 표시, hover로 전체 |

### 2.4 등급 필터 칩

카드 위에 필터 버튼:

```
[전체 38] [강력 8] [BUY 22] [WATCH 8]
```

- 기본: 전체 표시
- 칩 클릭 시 해당 등급 종목만 있는 섹터 카드만 표시
- `buy_grade` 매핑: `STRONG` → "강력", `BUY` → "BUY", `WATCH` → "WATCH"

### 2.5 정렬 규칙

1. **섹터 카드 순서**: `quant_sector_fire.fire_score` 내림차순
2. **카드 내 종목 순서**: `quant_sector_picks.buy_score` 내림차순
3. **종목 0개 섹터**: 카드 미표시

## 3. 데이터 조회 (기존 API 활용)

### 3.1 섹터 FIRE 데이터

```javascript
// quant_sector_fire — 최신 날짜
const { data: fires } = await supabase
  .from('quant_sector_fire')
  .select('sector, fire_score, fire_grade, fgn_5d, inst_5d, pension_5d, etf_code, etf_name')
  .eq('date', latestDate)
  .order('fire_score', { ascending: false })
```

### 3.2 종목 매수후보

```javascript
// quant_sector_picks — 최신 날짜
const { data: picks } = await supabase
  .from('quant_sector_picks')
  .select('*')
  .eq('date', latestDate)
  .order('buy_score', { ascending: false })
```

### 3.3 프론트엔드 그룹핑

```javascript
// sector 기준 그룹핑
const grouped = {}
picks.forEach(pick => {
  if (!grouped[pick.sector]) grouped[pick.sector] = []
  grouped[pick.sector].push(pick)
})

// fire_score 기준 섹터 정렬
const fireMap = {}
fires.forEach(f => { fireMap[f.sector] = f })

const sortedSectors = Object.keys(grouped)
  .filter(s => fireMap[s])  // FIRE 데이터 있는 섹터만
  .sort((a, b) => (fireMap[b]?.fire_score || 0) - (fireMap[a]?.fire_score || 0))
```
