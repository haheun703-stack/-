## 2. 목표 (P0 가설)

MarketRegimeGate가 BEARISH 검출 시:
- **현재 (5/20 가동)**: KILL_SWITCH 자동 활성화 → 매수 차단
- **개선 (6/15+)**: KILL_SWITCH **대신** → 후보 풀을 **인버스 화이트리스트로 전환** → 안전선 통과 시 인버스 1주 자동 매수

검증 가설:
- H1: 약세장(KODEX 인버스 +3%+)에서 인버스 ETF는 +4~+8% 수익
- H2: 인버스 ETF는 15:20 강제 청산 시점에 BEARISH 유지되면 양봉 마감 가능성 高
- H3: 사장님 룰 ① -3% 손절은 인버스에도 동일 적용 (변동성 高 시 빠른 회수)

## 4. 안전성 점검 (필수)

### 4-1. 인버스 ETF 위험 (정직 명시)

- **음의 복리**: 인버스2X는 2일 +1% / -1% 변동 시 원금 -0.02% 감소 (단기용)
- **변동성 高**: 일일 변동 ±5~10% (시장 -2.5%면 인버스2X +5%)
- **추세 반전 시 큰 손실**: 시장이 약세장 → 강세장 반전 시 인버스 빠르게 -5~-10% 하락 가능
- **거래량 高 필요**: 252670 일일 거래대금 수십억~ → 1주 매수 슬리피지 영향 ↓

### 4-2. 화이트리스트 가드

- `AUTO_TRADING_WHITELIST_ONLY=1`로 강제 (현재 사장님 결단으로 =0, 인버스 전환 시 conditional ON)
- 인버스 화이트리스트 외 종목 매수 0건 보장
- 정상 시장 (BEARISH 아님) 시 화이트리스트 OFF (현재 후보 풀 사용)

### 4-3. KILL_SWITCH 동시 사용

- MarketRegimeGate BEARISH → 인버스 전환 (KILL_SWITCH 안 만듦)
- 다른 5명 워커 FAIL → KILL_SWITCH 자동 활성화 (매수 0건)
- 즉 인버스 매수도 다른 안전망 통과해야 발화
