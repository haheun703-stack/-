## 0. 한 줄 요약

> 환율 상승·SpaceX IPO발 유동성 이동의 **원인 규명은 불가능하다(상관≠인과).**
> 퀀트봇이 할 일은 원인 단정이 아니라 **유동성 이탈의 증상을 정량 감시**하고,
> 위험 누적 시 **보조 veto(D0 금지/LIVE 보류)** 로 연결하는 것 — 전부 6/8 설계서의 실행이다.

## 2. 원칙 (기존 검증 결론과의 정합 — 변경 없음)

- **거시 = 시나리오 지도(맥락) / C60 = 핸들(타이밍)** (6/3 적대검증 결론)
- 환율·금리·유동성 어떤 것도 **단독 veto 금지, hard gate 승격 금지**
- "환율 상승 → 주가 하락"도 **전제가 아니라 검증 대상** (6/8 FX beta 실측: R² 7~14% = 환율은 변동의 일부만 설명, 5/29~6/1 환율↑ 속 현대차 +10.8% 반례 존재)
- 실주문 0 · freeze · scheduler/SAJANG 무변경 · **6/12 FLOWX 판정과 변수 분리** (본체 오염 금지)

## 3. 작업 항목

### P0 — 즉시 가능 (6/10~12, 데이터·read-only 분석만, FLOWX 본체 무접촉)

**P0-1. four_signal 사망 원인 규명 + 재발방지 (★이것이 1순위, backfill은 그 다음)**
- **★왜 5/19에 죽었나부터**: BAT 미배선? 스크립트 에러? 소스 단절(정보봇 usd_krw stale 사고와 동일 계열)? — 6/8 설계서가 stale을 인지하고도 3주 방치된 것 = **자동 갱신 체계 부재의 입증**. 원인 안 밝히고 backfill만 하면 내일 또 죽는다.
- **재발방지가 산출물의 본체**: ①갱신 경로를 일일 파이프라인에 배선(또는 수동 갱신이면 그 사실·주기·담당을 명시 박제) ②**staleness 가드** — preflight에 macro 데이터 최신성 체크 추가, N일 초과 시 경고 (조용히 죽는 것 차단)
- 그 다음 backfill: us10y/usdkrw/kospi_k 5/20~현재 + US30Y 신규 (FRED `DGS30`, 기존 fetch 인프라 재사용)
- 환율 소스 이중화 검증: FDR/yfinance 당일 vs 정보봇 usd_krw 교차 (6/2~6/8 stale 사고 재발 방지) — **당일 환율 실측값 확인도 여기서** (본 지시서의 환율 수치는 6/8 실측이 마지막)
- 검증 기준: 최신일=직전 거래일 / 결측 0 / **갱신 주체·주기가 문서화됨 / stale 경고 실제 작동 확인**

**P0-2. 환율 신호 적대검증 (read-only 분석)**
- 질문: `usdkrw_change_pct_1d ≥ +1%` / `20일 신고가` 가 KOSPI·FLOWX 후보군의 익일~5일 하락을 실제로 선행하는가?
- 방법: 외국인순매도 NOISE 판명(6/3)과 동일 — **base rate(상시점등 체크) / precision / recall / 신호 1~2일 지연 강건성 / 2022 환율위기 vs 2026 구간 분리**
- ECOS 환율 장기 시계열 사용 (수십 년 확보 가능)
- **결과 정직 반영**: NOISE 판명 시 MacroRiskScore 내 환율 가중 하향·"참고" 강등. 신호 판명 시 임계값(+1%/20일 신고가) 데이터 정당화
- **★사전 등록 (데이터 고문 방지)**: 2026 구간은 초강세장(환율↑·주가↑ 동행)이라 선행성이 노이즈에 묻힐 수 있고, 2022는 단일 에피소드 — **"NOISE 또는 표본 불충분" 판정이 나오는 것 자체가 정상적인 답**이다. 그 경우 환율 컴포넌트 "참고" 강등 + 약세 전환 후 재검증 예약으로 종결. **신호가 나올 때까지 조건을 바꿔 재실행하는 것 금지** (6/1 검증불가환경 교훈과 동형 위험)
- 산출물: 검증 리포트 (docs/03-report/) — 사장님의 "환율 불안" 이 우리 매매에 실제 위험 조건인지에 대한 데이터 답변

### P1 — 6/12 판정 후 (관측 레이어 구현, 매매 영향 0)

**P1 공통 — 모듈별 판정 시한 (좀비 관측 방지, v2 신설)**

이미 가동 중인 관측 트랙 6개(FLOWX·KODEX헤지·macro forecast·가격축·순환변동치·market_alignment)에 5개를 더 얹는 것이다. **관측을 늘린다고 알파가 나오는 게 아니다(6/1)** — 각 모듈은 시한 내 **배선/연장/폐기** 판정을 받는다. 표본 부족 시 1회 연장 가능(사유 박제), 무단 연장 금지.

| 모듈 | 판정 게이트 | 시한 | 폐기 조건 |
|---|---|---|---|
| fx_beta_observer | 2022 백테스트: fx_beta 상위군이 실제 더 빠지고 더 반등했나 | 구현 후 1주 | 차별성 없음 → 폐기 |
| MacroRiskScore 패널 | score≥2가 실제 하락(falling knife)을 선행했나 사후검증 | 관측 4주 | 선행성 없음 → 컴포넌트 축소·폐기 |
| 유동성 지표 3종 | 급락일 사전 경고에 기존 지표 대비 추가 기여 있나 | 관측 4주 | 외인 순매도 등과 중복 → 통합·폐기 |
| 이벤트 캘린더 | 첫 이벤트(SpaceX IPO) D+10 완주 기록 | 첫 이벤트 종료 시 | 유지비용 > 정보가치 → 수동 메모 강등 |
| 달러·금 슬리브 | 상관 매트릭스 산출 = 일회성 분석 | 산출 즉시 종료 | 상시 관측 모듈로 남기지 않음 (포트폴리오 OS 결정 입력으로만) |

**P1-1. fx_beta_observer 구현** (6/8 설계서 §6.5 단계 1·2 실행)
- read-only 일일 계산·기록: 종목별 fx_beta(60일 회귀), 환율인질스코어, 회복/경고 신호
- 2022 환율위기(1400+) 백테스트: fx_beta 큰 종목이 실제로 더 빠지고 더 반등했나
- data_store/ 적재, FLOWX 라우팅·tier 무접촉

**P1-2. MacroRiskScore 관측 패널** (6/8 설계서 §8 단계 1 실행)
- 5컴포넌트(us10y/us30y/usdkrw/외국인 전기전자/지수 갭하락) 변화율 합성 → 일일 score 기록
- SHOW ME 리포트 패널 추가 (6/7 가격축 레이어와 동일 방식 — 라벨만, 분기 사용 0)
- P0-2 적대검증 결과로 컴포넌트 가중 보정

**P1-3. 유동성 증상 지표 3종 신설** ("돈이 빠지는가" 정량화 — IPO·환율·관세 등 원인 불문 공통 증상)
| 지표 | 소스 | 가용성 |
|---|---|---|
| KOSPI/KOSDAQ 일별 거래대금 추세 (20일 평균 대비) | pykrx | ✅ 즉시 |
| 외국인 누적 순매수 (시장·전기전자) | 보유 수급 데이터 | ✅ 즉시 |
| 고객예탁금·신용잔고 | 금투협 freesis | ⚠️ 신규 소스 — 자동화 가능성 조사 후 결정 |
- 산출물: `liquidity_watch` 일일 기록 + SHOW ME 패널 (관측만)

**P1-4. 유동성 이벤트 캘린더 + 이벤트 스터디**
- SpaceX IPO 일정 등 대형 유동성 이벤트를 config 캘린더에 등록 (FOMC·발표일정과 같은 틀, macro tracker P2와 합류 가능)
- 이벤트 D−10~D+10 윈도우의 유동성 지표·환율·지수 반응을 **사후 기록**
- **정직선**: 표본 1인 단일 이벤트는 통계 검증 불가 — "IPO 때문" 단정 금지, 기록·관찰만. 누적되면(여러 메가 IPO) 패턴 검증 가능

**P1-5. 달러·금 슬리브 데이터 확보** (포트폴리오 OS 1단계와 합류)
- USD/KRW·금(원화환산)·미국지수(원화환산) 일별 적재
- 원화 기준 상관 매트릭스: 환율 상승 국면에서 달러 자산의 쿠션 효과를 2022 데이터로 실측
- → 환율 불안에 대한 **방어 수단의 정량 근거** (전략 상관 매트릭스와 동일 작업에서 산출)

### P2 — 관측 누적 후 (승인 별건)

- MacroRiskScore 임계(≥2 D0 금지 / ≥3 LIVE 보류)가 실제 falling knife를 선행했는지 사후 검증
- 통과 + 사장님 승인 시에만 LIVE 7조건 **상위 보조 veto** 배선 (6/8 설계서 §5)
- **절대 hard gate 승격 금지** — C60 단독 원칙 불변

## 4. 6/12 판정과의 관계 + 착수 시점
