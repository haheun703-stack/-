## 2. 핵심 구현 상세

### 2-A. STRATEGY_KR 매핑 사전 — 지시서 그대로 채택 + 영어 식별자 보조 표시
```typescript
// 매매 내역 + 적중률 표 strategy 컬럼 동일 패턴
<div className="text-[14px] font-bold">{getStrategyKr(s.strategy)}</div>      // 메인 한국어
<div className="text-[11px] text-[#9CA3AF] font-mono">                         // 보조: 영어 + 카테고리
  {s.strategy} <span>({categoryLabel(categorize(s.strategy))})</span>
</div>
```

지시서 핵심 원칙 준수:
- 코드 식별자(strategy 값): 영어 유지 ✅
- 사용자 표시: 한국어 풀이 ✅
- 디버깅용: 영어 식별자 + 카테고리 작은 회색 보조 표시 ✅

- 또는: 자동 fallback (`STRATEGY_KR[s] ?? s`)으로 영어 식별자 노출 — 사용자 혼란 가능

### 7-B. 다른 탭 영어 약어 (FIRE/RSI/PER 등) — 별도 단원
지시서 line 252-258: 섹터 발화/모멘텀/실적 괴리/피보나치 등 다른 탭의 약어 한국어 풀이 → **별도 지시서로 분리 권장**. 각 탭 컴포넌트 별도 영향이라 단원 분리가 안전.

### 7-C. 등급 코드 (AAA/AA/A/BBB/BB/B) 의미 정의
현재 memo `등급:AA` 정도 노출. 등급 코드 산출 기준 (시가총액? PF? 외인소진율?) 알려주시면 사용자 노출 텍스트 개선 가능.
