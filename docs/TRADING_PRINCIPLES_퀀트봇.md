# 매매 원칙 적용 지시서 — 퀀트봇 (quantum-master, 스윙)

> 이 문서는 퀀트봇 프로젝트의 Claude 세션에 전달하는 **구현 지시서**다.
> 출처: ①김동엽(증권 34년차) 인터뷰 ②개인투자자 성공기(1천만→21억) 영상.
> 단타봇(예언자)용 원칙(손익비 게이트, 뇌동매매 필터, 서킷브레이커)은
> 별도 문서에 있다. 이 문서는 **스윙/포트폴리오 호흡의 원칙만** 담았다.
>
> 이미 전달된 `insight_signals` v0.2.0(뉴스 키워드·DART 임원 매수·역발상
> 수급, 관찰 모드)과 겹치지 않는 **추가분**이며, 기존 검수 순서(본체 검수 →
> 접합부 수정 → D002 백테스트 → forward 관찰)를 바꾸지 않는다.

## 전제

1. 모든 신규 규칙은 insight_signals와 동일한 원칙으로 도입한다:
   **관찰(기록만) → 비이벤트 기저선 대비 검증 → 승격**.
2. 매매 로직 수정은 검증 통과 후에만. 그 전까지는 리포트/로그 레벨.
3. 프로젝트 CLAUDE.md 규칙(클린 아키텍처, cron 정본, LOCK 경로)이 항상 우선.

---

## 규칙 A — 주식:현금 = 7:3 포트폴리오 상한

**원칙**: "7:3이 황금비율. 올라도 좋고, 떨어져도 3으로 주울 용기가 생긴다.
몰빵은 심리(시스템으로 치면 대응 여력)부터 무너뜨린다."

**규칙 정의**:

```
총 평가자산 중 주식 비중 ≤ EQUITY_MAX (기본 70%)
- 신규 매수로 70%를 초과하게 되면 주문 수량을 자동 축소하거나 거부
- 하락으로 현금비중이 늘면: 정상 (추가 매수 여력으로 해석)
- 상승으로 주식비중이 75%+ 되면: 리밸런싱 후보 리포트 (자동 매도는 금지)
```

| 파라미터 | 기본값 |
|---|---|
| EQUITY_MAX | 70% |
| 리밸런싱 알림 임계 | 75% |

**구현 지침**: 페이퍼 계좌 평가액 스냅샷이 이미 있을 것이므로, 매수 주문
생성부 직전에 비중 체크 함수 하나로 구현 가능. 우선 **관찰 모드**로
"이 규칙이 있었다면 오늘 주문이 어떻게 조정됐을지"만 로그.

## 규칙 B — 주도주 로테이션 (스윙의 뼈대)

**원칙**: "돈이 쏠리는 주도주에 타고, 돈이 빠지면 눌린 곳으로 이동.
단, 테마는 한 때다 — 끝물엔 미련 없이 나온다." (두 영상 공통)

**규칙 정의** (관찰 지표로 먼저 구현):

```
섹터별 자금 흐름 점수 = f(섹터 지수 상대강도, 기관+외인 순매수, 거래대금 증가율)
- 상위 섹터 = 주도 섹터 → 보유/신규 후보의 소속 섹터와 대조
- 주도 섹터 이탈 신호: 거래대금 피크 대비 급감 + 상대강도 하락 전환
```

**구현 지침**: insight_signals의 KIS 수급 어댑터(`kis_flow_client`)를 재사용해
섹터(또는 보유 종목) 레벨로 집계하는 확장이 가장 경제적이다. 산출물은
일일 리포트에 "주도 섹터 Top3 / 이탈 경고" 섹션 추가 정도로 시작.

## 규칙 C — 미국장 섹터 강약 → 국내 시나리오 (전일 밤 → 아침)

**원칙**: "새벽에 미국장 섹터 강약을 보고 국내 시장 시나리오를 짜서
7시 전에 예약주문을 걸어놓는다." (성공기 인물의 핵심 루틴)

**규칙 정의** (v1은 리포트만):

```
1. 미국 섹터 ETF 등락 수집 (XLK 기술, SOXX 반도체, XLE 에너지,
   XLF 금융, XLV 헬스케어, XLI 산업재 ...)
2. 강세/약세 섹터 → 국내 대응 섹터 매핑 테이블 (설정 파일로 관리)
   예: SOXX↑ → 반도체/소부장, XLE↑ → 정유/조선기자재 ...
3. 아침 리포트: "오늘 강세 예상 섹터 / 약세 예상 섹터 + 근거"
```

**데이터 소스 (무료)**: 야후 파이낸스 quote API 또는 stooq.com CSV
(`https://stooq.com/q/l/?s=xlk.us&f=sd2t2ohlcv&h&e=csv` — 키 불필요).
cron상 실행 시각은 미국장 마감 후인 KST 06:0x대가 자연스럽지만, 기존
06:10 단계와 겹치지 않게 **서버 스케줄 아는 쪽이 최종 결정**할 것.

**검증**: "강세 예상 섹터의 당일 실제 수익률 - 시장 평균"을 매일 기록,
n≥30에서 초과성과 있으면 스코어링 반영 검토.

## 규칙 D — 공통 원칙 2건 (단타봇과 공유)

**D-a. 마이너스 먼저 잘라라** (김동엽: "플러스 종목을 팔고 마이너스를 들고
가는 게 100% 잘못. 강한 놈을 남겨라"): 포트폴리오 축소/교체가 필요할 때
매도 우선순위 = ①손실 확대 중 ②손실 정체 ③수익. 현행 페이퍼 매도
로직의 우선순위가 반대라면 **이게 가장 시급한 교정 대상**이다 — 먼저
현행 로직을 확인하고, 다르면 관찰 모드로 두 방식의 성과를 병행 기록할 것.

**D-b. 테마는 한 때다**: 주도 섹터 이탈 신호(규칙 B) 발생 시 해당 섹터
보유 종목을 리포트에 "이탈 검토" 플래그. 자동 매도는 검증 전 금지.

## 참고 — 이미 커버된 원칙 (중복 구현 금지)

| 원칙 | 담당 |
|---|---|

## 롤아웃 순서

1. 규칙 D-a 현행 로직 확인 (코드만 읽으면 됨 — 즉시)
2. 규칙 A 관찰 모드 (비중 로그 한 줄 — 저비용)
3. 규칙 C 아침 리포트 (신규 cron 1개 — 미국장 데이터 수집)
4. 규칙 B 섹터 로테이션 지표 (kis_flow_client 확장 — 마지막)

각 단계는 독립적이므로 하나씩 검증하며 진행. 전부 리포트/로그 레벨이며,
매매 로직 변경은 각 규칙의 검증 기준(n≥30, 기저선 대비 우위) 통과 후.
