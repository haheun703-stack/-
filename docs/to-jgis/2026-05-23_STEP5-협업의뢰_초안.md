## 2. 정보봇에 의뢰하는 데이터 3대 영역

### 2-A. STEP 3의 비용 데이터 — `quant_company_card` 확장 ★★★ (최우선)

**현재 갭**: 봇은 P(가격) + Q(거래량)만 측정. C(고정비/변동비) 완전 부재.

**의뢰 요청**: DART 재무제표에서 4분기 평균 비용 구조 추출.

```sql
-- 정보봇 측 quant_company_card 컬럼 추가 (예시)
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS fixed_cost_4q_avg BIGINT;
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS variable_cost_4q_avg BIGINT;
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS gross_margin_pct REAL;
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS operating_margin_pct REAL;
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS op_leverage_score INT;  -- 1~5 (영업 레버리지)
```

**활용**: 퀀트봇 realtime_score에 11번째 시그널 "비용 효율성" 추가 검토.

---

### 2-B. STEP 2 경쟁구조/경쟁력 점수 ★★

**현재 갭**: company_card에 biz_summary가 있으나 정량 점수 부재.

**의뢰 요청**: 정보봇 Perplexity 기반 정성 분석을 1~5 점수로 압축.

```sql
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS competition_intensity INT;  -- 1(독점) ~ 5(레드오션)
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS competitive_advantage INT;  -- 1(미흡) ~ 5(절대우위)
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS market_tam_billion KRW;     -- TAM (Total Addressable Market)
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS market_sam_billion KRW;     -- SAM (Serviceable Addressable Market)
```

**활용**: STEP 2 확장성 = (시장규모 × 경쟁력) ÷ 경쟁강도 자동 계산.

---

### 2-C. STEP 1 효용/인프라 점수 ★

**현재 갭**: theme_dictionary와 sector_fire가 있으나 종목별 효용 점수 없음.

**의뢰 요청**:

```sql
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS utility_score INT;          -- 1~5 (제품/서비스 효용)
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS price_competitiveness INT;  -- 1~5 (가격 경쟁력)
ALTER TABLE quant_company_card ADD COLUMN IF NOT EXISTS infrastructure_depth INT;   -- 1~5 (유통/생산 인프라)
```

**활용**: STEP 1 산업존재 = 효용 × 가격경쟁력 × 인프라 → 진입 게이트 보조.

## 3. 신규 테이블 제안 — `intelligence_step5_summary` (선택)

기존 `quant_company_card` 확장이 부담스러우면 **별도 테이블 분리**도 가능.

```sql
CREATE TABLE IF NOT EXISTS intelligence_step5_summary (
    ticker VARCHAR(10) PRIMARY KEY,
    name VARCHAR(100),

    -- STEP 1
    step1_utility INT,
    step1_price INT,
    step1_infra INT,
    step1_total REAL,        -- 가중 평균

    -- STEP 2
    step2_market_size_billion BIGINT,
    step2_competition INT,
    step2_competitive_edge INT,
    step2_total REAL,

    -- STEP 3
    step3_revenue_4q BIGINT,
    step3_fc BIGINT,
    step3_vc BIGINT,
    step3_op_margin REAL,
    step3_total REAL,

    -- STEP 4 (정보봇이 산출 가능한 부분)
    step4_earnings_persistence INT,  -- 1~5 (최근 8분기 영업이익 추세)
    step4_predictability INT,        -- 1~5
    step4_total REAL,

    updated_at TIMESTAMPTZ DEFAULT NOW(),
    source VARCHAR(50)  -- 'perplexity' | 'dart' | 'manual'
);
```

**장점**:
- quant_company_card 스키마 영향 X
- 정보봇이 독립적으로 갱신 주기 관리 (예: 주 1회 일요일)
- 퀀트봇은 read-only로 활용

## 4. 갱신 주기 제안

| 데이터 | 갱신 주기 | 트리거 |
|---|---|---|
| STEP 1~2 정성 점수 | **분기 1회** | DART 정기공시 발표 후 |
| STEP 3 재무 (비용/마진) | **분기 1회** | DART 사업보고서 + 분기보고서 |
| STEP 4 이익 지속성 | **분기 1회** | STEP 3과 동기 |
| TAM/SAM | **반기 1회** | Perplexity 산업 리포트 |

→ 퀀트봇은 매주 일요일 batch에서 한 번씩 fetch.

## 5. 우선순위 — 단계별 진행

### Phase 1 (5/27 ~ 6/15): MVP
- **2-A 비용 데이터만 우선** (DART에서 자동 추출 가능)

### Phase 3 (7월 이후): 미래시총 협업

- 5년 후 추정 시총 / 현재 시총 = 업사이드 비율
