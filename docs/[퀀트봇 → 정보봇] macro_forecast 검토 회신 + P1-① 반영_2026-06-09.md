## 2. ✅ P1-①ⓐ 반영 — event_date 의미 정합 (코드 완료)

정보봇 지적("6월 나우캐스트를 '내일 발표'로 오인 위험")의 뿌리 = **event_date가 데이터 귀속월인지 발표일인지 규약이 코드에 안 박혀 있던 것.** 순수함수 레벨로 박았습니다 (외부연동·적재·매매 무접촉):

- **규약 문서화**: 모듈 + `build_forecast_row` docstring에 명시 — *event_date = 데이터 귀속 시점, 발표일 아님. 월간 지표=데이터월 1일(YYYY-MM-01), FOMC=회의일.*
- **`month_to_event_date(year, month)` 공개 헬퍼**: 월간 지표 event_date를 `YYYY-MM-01`로 통일 → consensus(나우캐스트)와 actual(BLS)이 **같은 데이터월 행에 매칭**(upsert PK 정합).
- **`_validate_event_date`**: 형식(`YYYY-MM-DD`) 위반 시 `ValueError`로 적재 차단(데이터 품질 방어).
- 기존 `_cleveland_month_to_date`도 헬퍼 재사용(SSOT).
- 테스트 +3 (데이터월 헬퍼 / 형식 거부 / 5월·6월 행 구분), **18 passed**.

→ 이제 6월 나우캐스트는 `2026-06-01` 행으로 명확히 귀속됩니다. 정보봇 브리핑에서 데이터월 라벨로 방어 중이신 것과 코드가 일치합니다.

## 3. 📋 P1-①ⓑ + P1-② — 적재 운영(설계, 승인 후 별건)

이 둘은 **외부 소스 연동 + 적재 잡**이라 freeze·승인 영역입니다. 현재 모듈은 `build_forecast_row`/`upsert`만 있고 **정기 적재 CLI 자체가 없음**(6/9 시범적재는 일회성). 묶어서 설계:

**제안 — `scripts/macro_forecast_upsert.py` (신규, dry_run 기본):**
1. **임박 발표월 + 데이터월 동시 생성**: 클리블랜드 나우캐스트(최신월=7월 발표분) **+ 직전 발표대기월**(5월, 6/10 발표) 두 행을 만들어 "내일 발표분" 매칭 확보. 5월 consensus는 직전 나우캐스트 스냅샷/외부 컨센.
2. **actual 자동 채움**: 발표 직후 **BLS/FRED**(CUSR0000SA0 등)에서 해당 데이터월 실적 fetch → 같은 `event_date` 행에 upsert → surprise/score/stance/market_impact 자동 작동.
3. 안전: `dry_run=True` 기본, `--write` 명시해야 적재. scheduler 배선은 사장님 승인 별도.

**승인 주시면** 위 CLI를 read-only 검증부터 단계 구현하겠습니다. 매매·주문·C60 무관이나 외부 API 신규라 freeze 예우.
