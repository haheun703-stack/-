## 1. 배경 (정보봇 §3 인계)
- 보험·은행은 FCF 개념이 일반기업과 달라(보험영업현금흐름·예대 구조) yfinance `freeCashflow`가 무의미/왜곡됩니다.
- 실측 왜곡: **삼성생명 fcf_yield −57.8 / 삼성화재 −26.3**. 그대로 웹 노출 시 "현금 태우는 중"으로 **오해 소지**.
- verdict엔 무해(해당 종목 전부 ROE<15 → 가치함정 미트리거·관망)지만, 표시 숫자가 오해를 부름.

## 2. 퀀트봇 조치 — 소스에서 null 처리 (완료)
- `valuation_band.fetch_kr`에 `FCF_NA_TICKERS`(보험·은행·금융지주 8종목) 추가 → 해당 종목 `fcf=None` → `fcf_yield=None`으로 적재.
  - 대상: KB금융·신한지주·하나금융지주·메리츠금융지주·우리금융지주·기업은행 + 보험(삼성화재·삼성생명).
- **Supabase 6/17 KR 30행 재적재 확인**: 삼성화재·삼성생명 `fcf_yield = null`, verdict = 관망(불변).
- verdict 분포 불변(관망22/이미오름3/저점후보2/가치함정3) = **오분류 0 유지**(정보봇 검증과 정합).

## 3. 웹봇 측 = 조치 0 (권고)
- 이미 `fcf_yield`를 null-graceful 처리 중이면 **추가 작업 불필요** — 보험·금융 행은 FCF 칸이 "—"/N/A로 표시됩니다.
- (선택) 보험·금융 행 FCF 칸에 "업종 특성상 미산출" 툴팁을 달면 더 친절하나, 필수 아님.
- 만약 raw FCF 값이 다른 용도로 필요하면 알려주세요(현재는 데이터 정합성 위해 소스 차단).
