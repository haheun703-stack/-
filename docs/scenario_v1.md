# 우리만의 시나리오 v1 — 레짐 기반 선정·행동 기준 (2026-07-07 퐝가님 지시)

> "수급 기준으로만 가니까 한 번에 빠지면 우리만 손해. 우리만의 시나리오와 기준이 있어야 한다." (7/7)
> 본 문서는 감이 아니라 **이날까지의 백테스트 판정만으로** 조립한 선정·행동 기준이다.

## 1. 종목 선정 3층 구조 (모든 층은 백테스트 생존 근거 필수)

| 층 | 질문 | 도구 | 근거 |
|---|---|---|---|
| ①자격 (가치·성장) | 싸고 + 좋아지는 기업인가 | FV 엔진 (저PER+역사PER밴드+실적가속+수주) | KR 밴드 스프레드 +0.87~0.95%p·저PER 승률 67% |
| ②타이밍 (사이클·레짐) | 지금 이 종목·이 시장의 국면인가 | 주도주 사이클(초·중반만) + KOSPI/KOSDAQ 레짐 | 사이클 tolerance=6 검증·레짐 방어 실증(-12% 장서 페이퍼 -2~-4%) |
| ③확신 (수급 확인) | 스마트머니가 확인해 주는가 | 3단계(금투+연기금→기관→외인 마지막) | 3단계 D+1 +2.02%·WR59% — **단 BULL/정상장 한정** |

## 2. ★수급의 역할 재정의 (7/7 백테스트 2건의 공통 결론)

- **약세장 선행지표 ❌**: 약세일 기관 매집 상위 20% → D+20 **-2.22%p (t=-9.6)** 역효과 (bear_accumulation_report). 연기금 매수엔 기계적 리밸런싱이 섞여 정보가 아님.
- **강세장 확인지표 ✅**: 수급선행 초과수익은 강세 구간에서만 유효 (limit_up_precursor_report).
- 따라서 ③층(수급)은 **레짐이 BULL일 때만 발동**한다. BEAR에서 수급으로 종목을 고르지 않는다.

## 3. 레짐별 행동표

| 레짐 | 모드 | 행동 |
|---|---|---|
| **CRISIS** (MA60 아래) | DEFENSE | 신규 진입 0. 현금. 워치리스트 관측만 |
| **BEAR** (MA20~MA60) — *지금* | OBSERVE | 매수 없음. **워치리스트 = FV 자격 상위**(수급 무관) 매일 축적·관찰. 손절 규칙만 가동 |
| **CAUTION** (MA20 위+고변동) | PREPARE | 워치리스트 중 사이클 초·중반 통과 종목 후보화. 소규모 페이퍼만 |
| **BULL** (MA20 위+저변동) | ENTRY_WINDOW | 후보(FV∩사이클) 중 **3단계 수급 확인된 종목만** 진입. 주도주 추세 존중(눌림목 역추세 금지 유지) |

- BULL 판정기는 현행 V0(rv<50)이 기본. **V3b(하방변동<60, CAGR 18.9% vs V0 9.8%)가 승격 후보** — 시나리오 러너가 V0/V3b를 병행 산출·기록해 divergence를 축적하고, 퐝가님 승인 시 교체(brain_bull_relax_report).

## 4. 구현 (관측 전용 · 실주문 0 · freeze 무관)

- `scripts/run_scenario_v1.py` (BAT-D G5.8): 매일 레짐(V0+V3b 섀도)·모드·워치리스트(FV 상위10)를 `data/shadow/scenario_v1.json`에 기록.
- 승격 경로: BULL 전환이 실제로 오면 → 워치리스트→후보→진입 파이프라인의 forward 성과를 shadow로 검증 → 통과 시 페이퍼 배선 (퐝가님 승인).
