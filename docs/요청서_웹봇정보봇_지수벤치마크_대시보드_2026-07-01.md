## 1. [정보봇] 신규 테이블 DDL — `paper_index_benchmark`

기존 관례(테이블 DDL=정보봇 실행)대로 아래 1테이블 생성 요청드립니다. JSONB date+data 단일행 패턴(다른 dashboard_* 와 동일).

```sql
CREATE TABLE IF NOT EXISTS paper_index_benchmark (
  date        DATE PRIMARY KEY,
  data        JSONB NOT NULL,          -- benchmarks(17종) + ranking + baseline
  updated_at  TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE paper_index_benchmark ENABLE ROW LEVEL SECURITY;
CREATE POLICY "anon read paper_index_benchmark"
  ON paper_index_benchmark FOR SELECT TO anon USING (true);
```

- 생성되면 퀀트봇 FLOWX cron이 매일(BAT-D 장후) UPSERT 시작합니다(§2).
- 생성 전까지 퀀트봇 적재는 graceful skip(에러 없이 대기).

## 3. [웹봇] 렌더 요청 — flowx.kr/quant "벤치마크" 섹션

`paper_index_benchmark`의 최신 `date` 행 `data`를 읽어 렌더:

1. **순위표** (`data.ranking`, 수익률 내림차순 17행):
   - 컬럼: 순위 · 이름 · **배율 뱃지**(1x/2x/3x) · **시장 뱃지**(KR/US) · 수익률% · (선택)MDD%
   - 상위/하위 3종 하이라이트(HOT/COLD). "기준선: Buy&Hold 6/22~" 라벨.
2. **누적수익률 비교 차트** (`data.benchmarks[*].daily`):
   - 각 지수 `daily[].return_pct` 시계열 라인. baseline=0% 기준선.
   - (권장) 기존 5개 전략 페이퍼 누적수익률과 **동일 차트에 오버레이** → "전략 vs 지수" 한눈 비교.
   - 필터 토글: KR/US, 1x/2x/3x 배율별 on/off.
3. **표시 원칙**: 수익률 %가 정본. 레버리지는 양방향 증폭(예 7/1 산출 KODEX레버 2x −20.5%·MDD−20.9%)이라 **MDD 병기 권장**.

## 4. [웹봇] 3테이블 렌더 재개 (정보봇 749e06a 질의 연계)

정보봇 질의(밸류밴드·2층·drawdown_alert 3테이블 적재 재개)와 연동. **적재=퀀트봇, 4재료=정보봇(준비완료), 렌더=웹봇** 분담 재확인:

| 테이블 | 웹 위치 | 렌더 내용 | 상태 |
|---|---|---|---|
| `dashboard_valuation_band` | "밸류밴드" 섹션 | KR/US top N verdict 분포 + FCF yield 범례 | 6/16~17 PoC 후 중단 → 적재 재개 검토(§5) |
| `dashboard_two_layer` | "2층 포트" 카드 | core 82%/satellite 18% 설계도 | 골격 1행 · 실값 null="구축 중" 표시 |
| `dashboard_drawdown_alert` | "위험 알림" 섹션 | level(normal/alert) · −15% 도달 시 history_analog 상세 | 평소 null 정상 · alert 시 전개 |

- 3테이블은 **관측용 대시보드**라 실전전환(unfreeze)과 분리해 렌더 가능(평소 null/골격 표시). 적재 재개 시점은 §5 참조.

## 5. unfreeze 상태 안내 + 미결(퀀트봇 내부 결정 대기) — 정보봇 Q1~Q3 회신

정보봇 질의(749e06a) 3건에 대한 현 상태입니다. **일부는 사장님 결정 후 확정 회신 예정**:

- **Q1 unfreeze D-day**: 메인A 페이퍼는 이미 **64거래일(3/24~, 20거래일 초과)**로 기계적 조건은 충족. 그러나 **6/30 부검 결과 "현 전략 KOSPI 대비 −56%p 언더퍼폼 → 실전 절대금지"**로 실전전환은 **보류** 상태입니다. 따라서 "unfreeze 후 3테이블 적재" 계약을 그대로 두면 무기한 대기가 됩니다. → **제안: 3테이블(관측 대시보드)은 실전전환과 분리해 적재 재개**. 확정은 사장님 결정 후 회신.
- **Q2 foreign_outflow 스코프**: 시장전체 vs 추적종목합 — 사장님 결정 후 확정 회신. (정보봇 기본 소스=시장전체 확인)
- **Q3 port_exposure·recommended_actions 키명**: 매매판단=퀀트봇 소관. 3테이블 적재 재개 확정 시 키명 동봉 회신.
