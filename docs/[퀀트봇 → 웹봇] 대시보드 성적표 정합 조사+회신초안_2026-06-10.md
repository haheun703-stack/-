## 4. 회신 내용 (웹봇에 전달)

1. **웹 수정 = `?? 0` 제거가 아니라 컬럼명 교정**: `board.loss_count`→`board.lose_count`, `board.avg_return`→`board.avg_return_pct`. 이래야 525·0.87이 화면에 뜸. `?? 0`만 제거하면 undefined→null 될 뿐.
2. **migration이 SSoT 아님 — 실DB가 SSoT**(`select('*')`가 받는 것). migration의 loss_count/avg_return은 drift된 오류 → 별도 정리 필요.
3. **퀀트봇 몫 = 데이터 오류 아님**: signals·scoreboard 값 정상. 퀀트봇 작업 = scoreboard 갱신주체 재소유 + SSoT 매핑 박제(별도 트랙).

## 5. SSoT 매핑 (close_reason → 성공/실패) — ★사장님 결정 대기

| close_reason | 건수 | 평균 | 분류(퀀트봇·웹봇 공동권고 ⓐ) |
|---|---|---|---|
| TARGET_HIT / TARGET | 900 | +25.3% / +9.0% | ✅ 성공 |
| STOP_HIT | 663 | −8.47% | ❌ 실패 |
| TIME_LIMIT / SAR_REVERSAL | 692 | +0.58% / +2.46% | ⚠️ **return_pct 부호 판정(ⓐ)** |
| OPEN | 90 | — | 진행중 → 집계 제외 |

- **ⓐ(손익부호 판정)** = 퀀트봇·웹봇 공동권고(ⓑ 제외는 30% 버려 "불리한 것 뺐다" 의혹).
- 웹봇 추가요청: **return=0(정확히 무승부) 처리 규칙 명시**(제외 권장).
- 승률 = 성공/(성공+실패), OPEN 제외. 평균 = avg(return_pct), 미산출 시 **null**(0.0 금지).
- **★사장님 확정 필요: ⓐ로 확정? return=0 제외?**
