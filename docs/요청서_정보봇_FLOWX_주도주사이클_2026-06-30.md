## ✅ 요청 1 — `quant_leader_cycle` 테이블 생성 (DDL) ★필수

퀀트봇이 매일 BAT-D(16:30)에 주도주 90종목 사이클 진단을 이 테이블로 업로드합니다.
**현재 테이블이 없어 업로드가 매일 graceful 실패 중**입니다. 아래 DDL만 실행해주세요.

```sql
CREATE TABLE public.quant_leader_cycle (
  date date PRIMARY KEY,
  data jsonb NOT NULL,
  updated_at timestamptz DEFAULT now()
);

-- RLS: 기존 quant_* 테이블(quant_sector_fire 등)과 동일 정책 복제 (anon SELECT 허용)
ALTER TABLE public.quant_leader_cycle ENABLE ROW LEVEL SECURITY;
CREATE POLICY "anon read quant_leader_cycle" ON public.quant_leader_cycle
  FOR SELECT TO anon USING (true);
```

- 패턴은 `quant_market_brain` / `quant_sector_momentum`과 100% 동일(JSONB date+data 단일행).
- 생성되면 **그날 BAT-D부터 자동 적재** 시작(퀀트봇 추가 작업 불필요).

### 데이터 계약 (`data` JSONB 구조)

```jsonc
{
  "generated_at": "2026-06-30T11:05:00",
  "source": "config/global_leaders.yaml",
  "params": { "kr_tolerance_weeks": 6, "us_tolerance_weeks": 10 },
  "summary": {
    "total": 90, "us": 30, "kr": 60,
    "by_signal": { "청산": 34, "경계": 14, "매수적기": 12, "보유": 5, "해당없음": 23, "데이터없음": 2 }
  },
  "leaders": [
    {
      "market": "KR",            // "KR" | "US"
      "ticker": "012450",
      "name": "한화에어로스페이스",
      "sector": "방산",
      "signal": "청산",          // 매수적기 | 보유 | 경계 | 청산 | 해당없음
      "clock": "마감",           // 오전 | 정오 | 오후 | 마감 | 사이클없음  (생존율곡선 시계)
      "age_months": 26.0,        // 정배열 사이클 나이(개월)
      "survival_pct": 0.05,      // 잔여 생존확률
      "cycle_start": "2024-04-19",
      "delta_value": -37.9,      // 영업이익 성장률 델타(둔화=음수)
      "delta_source": "DART",    // DART(KR분기TTM) | US-annual | none
      "confidence": "high",      // high | mid | low
      "trend_intact": false,
      "mdd_from_high": -0.255,
      "revenue_dependency": "..." // KR만(글로벌 대장주 매출의존도)
    }
    // ... 90종목
  ]
}
```

### UI 표시 권장 (선택)
- **신호 배지**: 🟢매수적기 / 🔵보유 / 🟡경계 / 🔴청산
- **사이클 시계**: clock(오전🌅/정오☀/오후🌆/마감🌙) + age_months + survival_pct
- 정렬: 청산·경계(age 큰 순) → 매수적기(age 작은 순)
- **주의 문구 필수**: "생존편향 기반 통계 — 신호는 별도 펀더멘털·수급 확인 필요. 매매 미반영(관측)."

## ✅ 요청 2 — 한미충격(kr_us_shock) FLOWX UI 표시

정보봇이 올린 한미충격을 퀀트봇이 받아 **regime_macro 섹션에 노출**시켰습니다(역방향: 정보봇→퀀트봇→FLOWX).
이미 데이터는 들어가 있으니 **UI에서 보여주기만** 하면 됩니다.

- 위치: `brain_data_upload.json` → `regime_macro.kr_us_shock`
  (또는 정보봇이 직접 보유한 원천 daily_intelligence.json[kr_us_shock_summary])
- 구조:

```jsonc
"kr_us_shock": {
  "loaded": true, "status": "ok",
  "intel_date": "2026-06-29",   // 정보봇 적재일
  "date": "2026-06-26",         // 충격 측정 기준일
  "kr_shock": 64.8, "us_shock": 33.3, "diff": 31.5,
  "verdict": "한국 더 취약",
  "drivers": ["고환율 153...", "...", "..."],
  "note": "SHADOW 관측 전용 — 레짐/macro_score/SHIELD 미반영(freeze)"
}
```

- UI: verdict + diff 게이지(KR vs US) + drivers 3줄. "관측(매매 미반영)" 라벨 권장.

## ℹ️ 참고 — 외국인자금(china_flow) 섹션 당분간 빈 데이터

- 퀀트봇의 외국인자금 출처(`nationality_signal.json`)가 **KRX 잠금(6/12~)으로 6/09 고정** 상태였습니다.
- 6/29까지 이 묵은 데이터가 "현재"로 올라가던 버그를 발견 → **stale 가드를 걸어 업로드 중단**(빈 데이터가 낫다).
- 따라서 `china_flow`(외국인자금) 섹션은 **새 소스 연결 전까지 비어 있습니다**(정상, 버그 아님).
- 근본 해결: 3봇 분업상 **외인 수급=단타봇 담당** → 단타봇의 외인 데이터로 재연결 예정(별도 협의).
