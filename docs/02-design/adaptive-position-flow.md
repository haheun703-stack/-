## 1. 6단계 흐름 정의

```
[ 1. 상승 Hold ]      추세 진행 중 — 가만히 들고 있음
        ↓
[ 2. 상방 선 설정 ]   최근 N일 최고가 자동 감지 (= 천장 라인)
        ↓
[ 3. 천장 -3% 트리거 ]  현재가 ≥ 천장 × 0.97 + 신선도(최근 5거래일 내 천장 도달)
                       → 자동 매도 (전량 또는 80%)
        ↓
[ 4. 조정 단계별 분할매수 ]
        - 천장 -10% 도달: 매도 현금의 30% 매수
        - 천장 -20% 도달: 매도 현금의 30% 매수
        - 천장 -30% 도달: 매도 현금의 40% 매수
        - 평단가 = (천장 × 0.9 × 0.3 + 천장 × 0.8 × 0.3 + 천장 × 0.7 × 0.4)
                 = 천장 × 0.79 → 약 -21% 평단
        ↓
[ 5. 받침 패턴 ]      아래꼬리 캔들 + 다음날 양봉 + 거래량 증가
                       → 본격 재진입 (자비스 안전선 9건 + STEP 5 ★★★ 확인)
        ↓
[ 6. 다시 1단계로 ] 사이클 반복
```

## 2. 핵심 산식 (수학적 증명)

### 2-A. 분할매수 평단가 효과 (MDD 감소)

천장 100원 → -40% (60원) → -20% (80원) 회귀 시나리오:

| 전략 | 평단가 | MDD | 80원 회귀 시 |
|---|---|---|---|
| 단일매수 (100원 한 번) | 100원 | -40% | **-20% 손실** |
| **분할매수 30/30/40** (100/80/60) | **78원** | **-23%** | **+2.6% 수익** |

→ **MDD 22%p 감소 + 손실을 수익으로 전환**.

### 2-B. 천장 -3% 매도의 합리성

천장 도달 후 평균 조정폭은 통계적으로 -15~-25%.
천장 -3%에서 매도하면 평균적으로:
- 조정 -15% 시: +12%p 추가 보호
- 조정 -25% 시: +22%p 추가 보호
- 다시 상승만 (천장 회귀): -3% 기회 손실 (수용 가능)

→ **기대값 양수** (조정 빈도 × 평균 보호 > 회귀 빈도 × -3%).

## 4. 3중 검증 매매 시스템

```
[ ① STEP 5 (장기 5년) ]   →  펀더멘털 ★★★ 통과한 종목만 풀에 등록
                                       ↓

                                       ↓

```

**3중 검증 통과 종목만 매매** = MDD 견디기 + 복리 수익 + 펀더멘털 안전망.

## 5. MVP 4단계 구현 명세

### MVP-1: 천장 감지 + -3% 매도 트리거
**파일**: `src/use_cases/adaptive_position_manager.py`
**기능**:
- 보유 종목의 **30일 최고가** 자동 감지 (천장 라인)
- 현재가 ≥ 천장 × 0.97 + 천장 도달 신선도 (최근 5거래일 내)
- 트리거 시:
  - 텔레그램 알림 (천장 가격, 현재가, -3% 도달)
  - 자동 매도 (옵션 `ADAPTIVE_AUTO_SELL=1`일 때만)
  - 매도 후 분할매수 큐 등록 (MVP-2)

**임계 (`.env` 동적)**:
- `ADAPTIVE_PEAK_LOOKBACK_DAYS=30` (천장 감지 기간)
- `ADAPTIVE_PEAK_TRIGGER_PCT=0.97` (천장 -3%)
- `ADAPTIVE_PEAK_FRESHNESS_DAYS=5` (천장 도달 신선도)
- `ADAPTIVE_AUTO_SELL=0` (1주차는 알림만, 2주차부터 자동매도)
- `ADAPTIVE_SELL_RATIO=1.0` (전량 vs 부분 매도)

### MVP-2: 분할매수 큐 (-10%/-20%/-30%)
**파일**: `src/use_cases/adaptive_buy_queue.py`
**기능**:
- 매도 후 `adaptive_buy_queue.json`에 대기열 등록
- 종목별 3개 지정가 매수 주문 (천장 -10%, -20%, -30%)
- 가용 현금의 30%/30%/40% 분배
- 매 30분 cron에서 도달 여부 확인 → 매수 실행

**저장 구조**:
```json
{
  "queue": [
    {
      "ticker": "067310",
      "name": "하나마이크론",
      "peak_price": 50000,
      "stages": [
        {"trigger_price": 45000, "weight": 0.3, "status": "pending"},
        {"trigger_price": 40000, "weight": 0.3, "status": "pending"},
        {"trigger_price": 35000, "weight": 0.4, "status": "pending"}
      ],
      "registered_at": "2026-05-26T14:30:00",
      "available_cash": 1500000
    }
  ]
}
```

### MVP-3: 받침 패턴 자동 감지
**파일**: `src/use_cases/support_pattern_detector.py`
**기능**:
- 아래꼬리 캔들 (`low_shadow / body ratio ≥ 2.0`)
- 다음날 양봉 (`close > open + ATR × 0.5`)
- 거래량 5일 평균 대비 **2배 이상**
- 3 조건 ALL 충족 → 받침 신호 텔레그램 알림

### MVP-4: 자동 재진입
**파일**: `src/use_cases/adaptive_reentry.py`
**기능**:
- 받침 신호 발생 + STEP 5 ★★★ 매칭 + 자비스 9 안전선 통과 시 자동 매수
- 진입 수량 = 평단가 × N주 (가용 현금 한도)
- 진입 후 다시 MVP-1 천장 감지 사이클로 회귀
