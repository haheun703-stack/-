## 3. ★ 진범 — "쌍끌이"가 아니라 "연속 누적 길이(과매집)"

`supply_demand` 꼬리표를 까보니 가설보다 한 단계 더 날카로운 게 나왔다. **연속매집이 길수록 손실:**

| 수급 꼬리표 | n | pnl |
|---|---|---|
| 쌍끌이매집 / 기관7연속+외인8연속 | 1 | **−9.99%** (최장·최악) |
| 쌍끌이매집 / 기관9연속 | 2 | −7.59% |
| 쌍끌이매집 / 기관7연속 | 1 | −7.67% |
| 쌍끌이매집 / 외인3연속 | 1 | −7.21% |
| 쌍끌이매집 / 기관4연속 | 1 | −6.39% |
| 외인매수 / 외인3연속 | 1 | −7.13% |
| **(연속꼬리 없는) 외인매수** | 5 | **+4.95%** ✅ |
| **(연속꼬리 없는) 쌍매도** | 5 | **+2.89%** ✅ |

**"N연속" 꼬리표가 붙은 9건 중 7건이 −6~10% 손실.** (예외 2건: 기관6연속 +12.78%·외인4연속 +8.57%.)

→ **메커니즘**: 합의가 완성될 때까지(연속 매집이 길어질 때까지) 기다렸다가 AA로 최우선 진입 = **"모두 이미 산 종목을 천장에서 산다" → 반전**. 쌍끌이 라벨 자체보다, 그 쌍끌이가 **이미 오래 누적된 상태(과매집)**일 때 진입하는 게 독이다.

### 3.1 정정 — AA 부진은 쌍끌이 단독 탓이 아니다
- **AA × (수급꼬리 없음) 18건도 −1.59%.**
- AA로 승격시키는 경로 전반(STRONG_ALPHA +50 부스트·score≥70 → 최우선 진입)이 천장을 잡는 경향. 쌍끌이/과매집은 그중 **가장 선명한 한 갈래**.

### 3.2 표본 한계 (정직히)
- 연속꼬리 세분은 n=1~2 → 개별 수치는 노이즈.
- 그러나 방향(연속길이↑ → 손실)은 9건 중 7건, 그룹 비교(A>AA n=18/45·외인단독>쌍끌이 n=5/15)는 메커니즘까지 일관 → "운"이 아니라 **로직 결함** 단계.

## 4. 재설계 원칙 — 확신 "반전"

기존: **합의 완성도(연속 누적) = 확신** → 반전:

| 신호 | 기존 처리 | 재설계 |
|---|---|---|
| 쌍끌이 + 장기연속(과매집) | **AA 강제 + 50점** | **감점** (천장 위험) |
| 외인 단독(선행 1주체) | 보통 | **가점** (선취매 = 초입) |
| 단기 쌍끌이(연속 짧음) | AA | 중립~소폭 가점 (초입 가능성) |
| score≥70 + 다소스 | AA | 유지하되 과매집 필터 통과 시만 |

### 4.1 핵심 — "과매집 감점 함수" (신규, 관측→게이트)
연속 누적 길이를 **선형 페널티**로:
- 외인/기관 연속 일수(`supply_demand` 꼬리표에서 파싱) → `crowding_penalty`
- 연속 ≤2일: 페널티 0 (초입)
- 연속 3~5일: 소폭 감점
- 연속 ≥6일(특히 기관·외인 동시 장기): 강한 감점 → AA 승격 차단

### 4.2 연결 — 기존 미배선 엔진 3종 활용
오늘 발견의 메커니즘은 이미 만들어 둔 엔진들과 직결된다:
- **`crowding.py`** (6/14, L3 모니터): C1 평균쌍상관·C3 외인선물 → **과매집 = crowding의 종목 단위 버전**. 같은 철학.
- **`sector_momentum_label.py`** (6/10): 약세섹터 과매집은 더 위험 → worst_regime 노출 감점 연결.
- **밸류밴드** (`valuation_band.py`, 6/15): "이미 오른" 천장 종목 = FCF 가치함정 교차검증.

### 4.3 설계 산식 초안 (의사코드, 미구현)
```
def conviction_score(pick):
    base = pick.score
    # (1) 과매집 감점 — 진범
    days = parse_consecutive_days(pick.supply_demand)  # "기관7연속+외인8연속" → 15
    if days >= 6:   base -= STRONG_CROWDING_PENALTY    # 천장
    elif days >= 3: base -= MILD_CROWDING_PENALTY
    # (2) 선행(외인단독·연속짧음) 가점
    if is_foreign_only(pick) and days <= 2: base += LEAD_BONUS
    # (3) 섹터·밸류 교차
    if sector_worst_regime(pick): base -= SECTOR_PENALTY
    if valuation_trap(pick):      base -= TRAP_PENALTY
    return base
# AA 강제 승격(L334)은 conviction_score 통과 후로 이동
```

## 6. ★ freeze / 매매로직0 불변식 충돌 검토

- 본 재설계는 **페이퍼 진입 전략(등급/사이징)** 변경 = 넓게 보면 "매매로직 변경".
- 단, **실주문 경로(KisOrderAdapter·리스크 게이트)는 무접촉**, 페이퍼 SCAN 후보 선정 로직만 수정.
- **판단 필요(사장님)**: ① freeze 베이스라인을 "페이퍼 전략 개선은 허용"으로 해석할지, ② unfreeze(서버배포 E-0 + 20거래일 + 흑자전환) 이후로 미룰지.
- **권고**: 엔트리 개선은 E-0 게이트검증(리스크 한도)과 **직교** → 페이퍼 한정·실주문 무접촉 조건이면 **병행 가능**. 단 구현은 본 설계서 승인 후, 별도 커밋·페이퍼 격리로.
