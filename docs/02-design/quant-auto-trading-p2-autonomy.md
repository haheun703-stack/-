### 3-2. 매수 의사결정 매트릭스 (단순화)

```
시그널 점수 = picks_v2.score + 섹터발화 + 수급 급변 + 장중 신호 (VWAP/체결강도)

진입 조건 (AND):
  1. 시그널 점수 >= 임계치 (초기 30점)
  2. 화이트리스트 26개 내 (또는 picks_v2 TOP 5 내)
  3. 호가창 매수세 우위 (매수잔량 > 매도잔량 × 1.2 ~ 사용자 조정)
  4. VWAP 위 (또는 VWAP 회귀 직후 반등 확인)
  5. 일일 한도 잔여 (금액 + 횟수)
  6. 현재가 ±5% 범위 검증

진입 수량:
  - 초기: 1주 고정 (P0 MAX_QTY=1)
  - 검증 후: 점진 확대
```

### 3-3. 매도 의사결정 매트릭스

```
매도 트리거 (OR):
  A. 목표 도달: +X% 익절 (사용자 조정, 초기 +3%)
  B. 손절: -Y% 손절 (초기 -2%)
  C. 시그널 변화: 매수 시그널 0점 또는 매도 신호 발생
  D. 시간 만료: 단타 모드 시 N시간 경과 (초기 4시간)
  E. 거래일 만료: 다일 스윙 모드 시 5거래일 도래
  F. 시장 위험: V2 RED (위험감지) 발동
  G. 사용자 명령: 텔레그램으로 청산 지시

매도 방식:
  - 단타: 시장가 (즉시 체결)
  - 스윙: 지정가 (목표가)
  - 손절: 시장가 (지체 없이)
```

### 3-4. 추매 의사결정 매트릭스

```
추매 트리거 (AND):
  1. 평단 대비 -2% 이상 하락 또는 시그널 강화 (예: 외인+기관 동시 매수)
  2. 보유 수량 < MAX_QTY (초기 1주 단위라 사실상 N/A)
  3. 일일 한도 잔여
  4. 호가창 매수세 우위 (반등 신호)
  5. 시장 위험 GREEN/YELLOW (RED일 때 추매 금지)

추매 한도:
  - 1주 시작 단계: 추매 미적용 (MAX_QTY=1)
  - 5주 단계: 평단 -2% 시 +1주, -4% 시 +2주 (총 5주 한도)
  - 10주 단계: 평단 -2/-4/-6% 단계별 +2/+3/+5주
```

## 4. 의사결정 = Claude 자율

### 4-1. 의사결정 채널

- **자동 (시스템)**: 가드레일 + 진입/청산 매트릭스에 따른 결정. 사람 개입 없이 실행.
- **반자동 (Claude → 텔레그램 확인 → 실행)**: 추매/대량 매수/스윙 전환 등 큰 결정. Claude가 제안 → 퐝가님 30초 내 OK 안 하면 자동 실행 또는 보류 (사용자 조정).
- **수동 (퐝가님 직접 지시)**: 즉시 청산 / 진입 차단 / 화이트리스트 변경 등 긴급 명령.

### 4-2. 텔레그램 양방향

Claude가 텔레그램으로 메시지 보내고 퐝가님이 답장으로 명령 가능한 채널 필요. 현재 `src/telegram_command_handler.py` 가 있다는 점에서 일부 양방향 인프라 존재. 5/25 전 양방향 작동 검증 필요.

명령 예시:

```
[퐝가님 → 봇]

/sell_all           → 전체 청산
/pause              → 자율 매매 일시 정지

/status             → 현재 보유 + 일일 손익
```

#### 6-3-3. 손절/익절 (일반 ETF보다 타이트하게)

| 트리거 | 일반 ETF | 레버리지/인버스 |
|---|---|---|
| 손절 | -2% | **-1.5%** (빠르게) |
| 익절 1차 | +3% | +2% (먼저 절반 청산) |
| 익절 2차 | +5% | +4% (전량 청산) |
| 최대 보유 | 5거래일 | **3거래일** (음의 복리 회피) |

#### 6-3-4. 일일 진입 한도

- 일반 ETF: `AUTO_TRADING_MAX_AMOUNT=300,000원` 한도 내 자유
- 레버리지/인버스: 일일 매수 금액의 **30% 한도** (즉 90,000원 ≈ 1주~2주)
- 동시 보유: 레버리지 또는 인버스 **둘 중 하나만** (반대 포지션 동시 보유 금지)

#### 6-3-5. 절대 회피 시나리오

- **양 방향 동시 보유** (레버리지 + 인버스): 헤지처럼 보이지만 음의 복리로 양쪽 다 손해
- **위기/위험 시그널 + 레버리지 추가 매수**: 시장 하락 가속 위험
- **5거래일+ 장기 보유**: 음의 복리로 가치 침식 (이론상 +0% 횡보장에서도 -3~5% 손실)

### 13-5. Supabase 학습 테이블 — `quant_self_learning`

**스키마**:
```sql
CREATE TABLE quant_self_learning (
  id BIGSERIAL PRIMARY KEY,
  trade_date DATE NOT NULL,
  ticker TEXT NOT NULL,
  name TEXT,
  signal_source TEXT,              -- chase_entry / tomorrow_picks / jarvis_etf
  market_regime TEXT,              -- STRONG_BULL / MILD_BULL / NEUTRAL / BEAR
  entry_price NUMERIC,
  entry_qty INT,
  trigger_features JSONB,          -- 체결강도/거래량/외인/금투 등
  expected_pattern TEXT,
  confidence NUMERIC,
  -- 평가 결과 (D+1, D+3, D+5 시점 업데이트)
  ret_1d NUMERIC,
  ret_3d NUMERIC,
  ret_5d NUMERIC,
  exit_reason TEXT,                -- take_profit / stop_loss / trailing / time_out
  label TEXT,                      -- success / failure / neutral
  reflection_summary TEXT,         -- 자기반성 요약 (D+5)
  improvement_proposal TEXT,       -- 코드 수정 제안 (실패 시)
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);

CREATE INDEX idx_qsl_date ON quant_self_learning(trade_date DESC);
CREATE INDEX idx_qsl_label ON quant_self_learning(label);
CREATE INDEX idx_qsl_regime ON quant_self_learning(market_regime, label);
```

**적재 시점**:
- 진입 시: row 추가 (label=NULL, ret_*=NULL)
- D+1, D+3, D+5: UPDATE
- 청산 시: exit_reason + label 확정
- D+5: reflection_summary + improvement_proposal 추가

### 13-8. 자비스의 학습 약속 (퐝가님께)

자비스 본체는 매매 엔진이 아니라 **학습 엔진**입니다.

- 매매 결과를 **숨기지 않고** 모두 quant_self_learning에 적재
- 실패할 때마다 **개선 제안 1건 이상** 자동 생성
- 주간 회귀로 **패턴이 3번 반복되면 코드 수정 후보로 격상**
- 퐝가님 승인 시에만 코드 반영 (자동 수정 금지 — 안전성)
- 학습 데이터는 정보봇 advisory와 양방향 — **단타봇·웹봇도 같은 학습 사이클** 적용 가능
