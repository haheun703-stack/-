## 2. 퀀트봇 페이퍼 기준

퀀트봇은 약할 때 사는 눌림/지지 반등형이다.

진입 후보:

- 대형주/소부장/섹터발화 후보
- 주봉 게이트 `LONG_ALLOWED`
- 일봉 눌림 후 지지 반등 확인
- 과열 아님
- 손익비 양호
- 수급은 보조점수만, hard gate 금지

paper 진입 기록:

- `T0 종가 진입`
- `T+1 시가 진입`
- 둘 다 병행 기록

매도/청산 기록:

- 지지선 이탈 손절
- MA10/MA20 이탈
- D+10 점검 기준
- MFE/MAE 필수 기록

## 3. 단타봇 페이퍼 기준

단타봇은 강할 때 타는 섹터/그룹 로테이션형이다.

3개 타입을 병행한다.

| 타입 | 내용 |
| --- | --- |
| A | STEADY + S급 DART/이벤트 라이드 |
| B | 강한 섹터/그룹 눌림 매수 |
| C | 강한 섹터/그룹 올라타기 |

공통 기록:

- 강한 섹터/그룹 상태
- RS
- 거래량
- 마감강도
- 이격도
- 뉴스/DART/event 명분
- 수급은 feature/log만, hard gate 금지

청산 기록:

- D+1/D+2/D+3
- MA10
- 구조 이탈
- 손절 variant 병행

## 4. 수급 사용 원칙

수급은 매수 방아쇠가 아니다.

- 시장 레벨: 11주체 세부 divergence 관측 가능
- 종목 레벨: 현재 4주체만 기록
- 종목별 11주체 판단 금지
- 기관합계 hard gate 금지
- 금융투자/연기금/외국인등록/기타외국인/개인은 분리 기록

## 5. 6/10 실전 전환 조건

아래 조건을 모두 충족해야 한다.

1. 실주문 경로 안전검수 PASS
2. paper ledger 정상 기록
3. 후보/진입/회피/매도 사유가 모두 남음
4. 최소 2~3건 이상 유효 샘플 확보
5. 손절/청산 기준이 코드/문서에 명확함
6. 수급 hard gate 미사용 확인
7. scheduler/systemd/SAJANG 무변경 확인
8. 사장님 최종 승인

## 6. 실전 전환 방식

조건 충족 시에도 6/10은 소액만 허용한다.

- 자동매매 전면 ON 금지
- paper와 병행
- 1회 주문/1일 한도 제한
- 손절 미정 종목 진입 금지
- 보고 없이 추가 매수 금지

## 7. 최종 원칙

6/10 실전은 목표가 아니라 옵션이다.  
6/9까지 데이터가 부족하거나 안전검수가 부족하면 실전 전환하지 않는다.
