기간: 2026-06-05 ~ 2026-06-09  
목표: 6/10 조건부 소액 실전 전환 가능 여부 판단

## 0. 최종 운용 철학

이번 페이퍼는 한 전략을 고르는 테스트가 아니다. 역할이 다른 두 엔진을 나눠서 검증한다.

| 엔진 | 역할 | 기대 |
| --- | --- | --- |
| 퀀트봇 | 밸류 바닥 + 주봉/일봉 반등 확인 | 성공확률, 물림 방지, 안정성 |
| 단타봇 | 섹터/그룹 로테이션 눌림·올라타기 | 수익금, 속도, 시장 쏠림 포착 |

예상:

- 수익률 1등 후보: 단타봇 C 로테이션 올라타기
- 안정성 1등 후보: 퀀트봇 밸류 바닥형
- 중간형 후보: 단타봇 B 로테이션 눌림

실전 판단은 "단타로 벌고, 퀀트로 방어한다"는 구조를 전제로 한다.

## 1.5 6/5 시작 전 선결조건

페이퍼 트레이닝은 데이터 기준일과 ledger가 정상일 때만 시작한다.

### 1.5.1 퀀트봇 선결

- 로컬 processed 지표 재동기화: 6/4 rebuild는 VPS 기준으로 완료됐으므로, 로컬 분석 전 VPS→로컬 processed 동기화 또는 로컬 재계산 확인 필수
- C60/regime_monitor 기준 확인: 488080·삼성·SK 셋 다 최신 6/4 기준 `BULL` 여부 재확인
- 유령행 재발 점검: 6/3 휴장 행, OHLCV=0 행 잔존 0 확인
- paper ledger 필드 확인: `T0_CLOSE`, `T1_OPEN`, `MFE`, `MAE`, 후보/진입/회피/매도 사유 기록 가능해야 함
- 주봉 게이트 확인: `LONG_ALLOWED / WATCH / AVOID`가 종목별로 저장되어야 함

### 1.5.2 단타봇 선결

- paper ledger 인프라 점검: A/B/C 타입별 독립 ledger, 자본/성과/MFE/MAE 분리
- `day_trade_bot.py` 프로토타입은 read-only 참고물로만 취급. 수급동행 hard gate 제거 전까지 실행/실데이터 연결/편입 금지
- 단타봇 수급은 `supply_score`, `flow_context`, `flow_warning` 같은 feature/log로만 기록
- HOT/WARMING/RELAY 그룹 분류와 RS/거래량/마감강도/이격도 필드가 빠짐없이 남아야 함

### 1.5.3 공통 금지

- 실주문 0
- scheduler/systemd 자동배선 금지
- SAJANG 변경 금지
- 데이터 복구 중 전체 파이프라인 연쇄 재실행 금지

## 2. 퀀트봇 페이퍼 기준 — 밸류 바닥형

퀀트봇은 약할 때 사는 밸류 바닥 + 눌림/지지 반등형이다.

진입 후보:

- 대형주/우량주/유동성 큰 소부장
- 밸류 바닥권: PBR 과거 2년 하위 10%권 또는 역사적 저점권
- EPS 양수 또는 EPS 급감 없음
- 개별 악재/실적 훼손 없음
- 거래대금 100억 이상 우선, 300억 이상은 가점
- 주봉 게이트 `LONG_ALLOWED`
- 일봉 눌림 후 지지 반등 확인
- 과열 아님
- 손익비 양호
- 수급은 보조점수만, hard gate 금지

퀀트봇에서 제외:

- 밸류는 싸지만 회사 자체 악재가 있는 종목
- EPS 훼손/적자 전환/업황 붕괴 종목
- 유동성 부족 종목
- 주봉 `AVOID`

paper 진입 기록:

- `T0 종가 진입`
- `T+1 시가 진입`
- 둘 다 병행 기록

매도/청산 기록:

- 지지선 이탈 손절
- MA10/MA20 이탈
- D+10 점검 기준
- MFE/MAE 필수 기록

퀀트봇 6/9 판정:

- 밸류 바닥형이 실제로 물림을 줄였는가
- 유동성 100억/300억 필터 차이가 있었는가
- PBR 저점 + EPS OK 조합이 일반 눌림보다 나았는가
- 개별 악재 제외가 손실 회피에 도움이 됐는가

## 3. 단타봇 페이퍼 기준

단타봇은 강할 때 타는 섹터/그룹 로테이션형이다.

3개 타입을 병행한다.

| 타입 | 내용 |
| --- | --- |
| A | STEADY + S급 DART/이벤트 라이드 |
| B | 강한 섹터/그룹 눌림 매수 |
| C | 강한 섹터/그룹 올라타기 |

공통 기록:

- 강한 섹터/그룹 상태
- RS
- 거래량
- 마감강도
- 이격도
- 뉴스/DART/event 명분
- 수급은 feature/log만, hard gate 금지

청산 기록:

- D+1/D+2/D+3
- MA10
- 구조 이탈
- 손절 variant 병행

단타봇 6/9 판정:

- C 올라타기가 실제 수익률 1등인지
- B 눌림이 C보다 안정적인지
- A S급 명분이 수익률/승률을 높였는지
- RS/거래량/마감강도/이격도 중 무엇이 가장 돈 됐는지

## 4. 수급 사용 원칙

수급은 매수 방아쇠가 아니다.

- 시장 레벨: 11주체 세부 divergence 관측 가능
- 종목 레벨: 현재 4주체만 기록
- 종목별 11주체 판단 금지
- 기관합계 hard gate 금지
- 금융투자/연기금/외국인등록/기타외국인/개인은 분리 기록

### 4.1 `foreign_exh` ghost 처리

`foreign_exh`는 현재가 스냅샷 API 기반이라 날짜/기준일을 잘못 찍으면 휴장·장전 ghost가 생긴다. 6/5 페이퍼 전 아래 기준을 적용한다.

원인:

- `foreign_exh`: 현재가 스냅샷 API + `datetime.now()` 날짜 사용
- `investor`: KIS 응답의 실제 거래일(`stck_bsop_date`) 사용
- 이 비대칭 때문에 6/3 휴장, 6/4 장전 수집 시 "오늘 날짜 + 직전거래일 값" ghost가 생성됨

즉시 데이터 처리:

- 6/3 휴장 ghost 행 제거
- 6/4 `foreign_exh`는 장마감 후 force 재수집으로 진짜 6/4 값 self-heal 가능
- 재수집 전 `foreign_exh`는 매매/페이퍼 진입 기준으로 사용 금지

근본 수정 방향:

- 휴장가드만으로 부족하다. 거래일 장전 수집도 ghost를 만들 수 있다.
- `foreign_exh`는 장마감 후에만 today 행 생성하거나, API 응답에 거래일 필드가 있으면 그 날짜를 사용한다.
- 코드 fix는 4-Tier 검수 후 별도 진행한다.

페이퍼 적용:

- `foreign_exh`는 수급 fact layer 중 하나일 뿐이다.
- data_date가 검증되지 않은 `foreign_exh`는 `stale_or_ghost_suspect=true`로 기록하고 점수에서 제외한다.
- 종목 진입 hard gate로 사용 금지.

## 5. 6/10 실전 전환 조건

아래 조건을 모두 충족해야 한다.

1. 실주문 경로 안전검수 PASS
2. paper ledger 정상 기록
3. 후보/진입/회피/매도 사유가 모두 남음
4. 최소 2~3건 이상 유효 샘플 확보
5. 손절/청산 기준이 코드/문서에 명확함
6. 수급 hard gate 미사용 확인
7. scheduler/systemd/SAJANG 무변경 확인
8. 사장님 최종 승인

추가 조건:

- 퀀트봇은 안정형 후보로만 소액 허용
- 단타봇은 C/B/A 중 paper 성과가 가장 명확한 타입만 소액 허용
- 두 봇 모두 같은 날 시장 위험이 높으면 실전 전환 보류

## 6. 실전 전환 방식

조건 충족 시에도 6/10은 소액만 허용한다.

- 자동매매 전면 ON 금지
- paper와 병행
- 1회 주문/1일 한도 제한
- 손절 미정 종목 진입 금지
- 보고 없이 추가 매수 금지
