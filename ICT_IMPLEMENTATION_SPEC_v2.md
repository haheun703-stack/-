# ICT 개념 적용 가이드 — 봇 구현 스펙

> **이 문서는 Claude Code 실행용입니다.**
> 원본 ICT 영상 분석 → 한국 시장 변환 → 3봇 적용 스펙을 Phase별로 정리합니다.
> Phase 1 완료 후 실전 검증 → Phase 2 착수 → 검증 → Phase 3 순서로 진행합니다.

---

## 배경: ICT에서 가져올 것 / 버릴 것

| ICT 개념 | 근거 수준 | 판정 |
|---------|----------|------|
| 시간대별 거래량 집중 (킬존) | ★★★★★ KRX 거래소 데이터로 검증 가능 | ✅ 채택 |
| Opening Range / Initial Range | ★★★★☆ 학술 연구 존재 (Crabel 1990) | ✅ 채택 |
| 프리미엄 유동성 레벨 (전일고/저, 주간고/저) | ★★★☆☆ 지지/저항의 체계적 변형 | ✅ 채택 (보완용) |
| Equal High/Low (동등 고점/저점) | ★★★☆☆ 다중 터치 레벨의 관찰적 유효성 | ⚠️ 조건부 채택 |
| FVG / HP FVG (Fair Value Gap) | ★★☆☆☆ 경험적, 정량 검증 부족 | ⚠️ Phase 3 |
| AMD 패턴 (축적-조작-분배) | ★★☆☆☆ 내러티브 중심, 사후 해석 위험 | ⚠️ Phase 3 |
| 스마트머니 음모론 | ★☆☆☆☆ 근거 없음 | ❌ 버림 |

**핵심 원칙**: 근거 ★★★ 이상만 Phase 1에 넣는다. ★★☆는 Phase 3에서 백테스트 후 판단.

## Phase 1 — 즉시 구현 (근거 ★★★★ 이상, 효과 확실)

### 모듈 1-1. 시그널 우선순위 상수화
- **대상**: 정보봇 (이벤트 트래커)
- **난이도**: ★☆☆
- **목적**: 현재 암묵적으로 작동하는 시그널 우선순위를 명시적 상수로 정의

```python
# signal_priority.py (신규)

SIGNAL_PRIORITY = {
    1: "터치 금지",              # 역배열 + 과매도 + 반전 미확인 (v2.1 기존)
    2: "프리미엄 레벨 경고",      # 전일고/저, 주간고/저 근접 시 주의
    3: "목표 달성 / 과매수",      # 목표가 도달 or RSI > 70
    4: "손절 경고",              # 손절가 근접
    5: "눌림목 (진입 대기)",      # 정배열 + 음봉 + 종가 > MA20 (v2.1 기존)
    6: "추세 약화 경고",          # 정배열 + MA20 이탈
    7: "중립",                  # 특이사항 없음
}

# 규칙: 숫자가 낮을수록 우선. 상위 시그널이 하위를 덮어씀.
# 예: 농심이 기술적으로 "눌림목(5)"이지만 "목표근접+과매수(3)"가 있으면 → 3 적용
```

- **구현 방법**: 기존 이벤트 트래커의 시그널 판정 로직에서 이 상수를 참조하도록 리팩터링
- **테스트**: 기존 4종목(한국콜마, 농심, 코스맥스, 파마리서치)에 대해 v2.1과 동일한 결과가 나오는지 확인

### 모듈 1-2. OR/IR 자동 기록
- **대상**: 퀀트봇 + 단타봇
- **난이도**: ★★☆
- **목적**: 09:00~09:30(OR), 09:00~10:00(IR) 구간의 고점/저점을 자동 기록하여 당일 기준점으로 활용

```python
# opening_range.py (신규)

"""
기능:
  1. 장 시작 후 OR 구간(09:00~09:30) 고점/저점 기록
  2. IR 구간(09:00~10:00) 고점/저점 기록 (OR 포함 확장)
  3. 당일 바이어스 판정:
     - 10:00 이후 OR 고점 돌파 → bullish bias
     - 10:00 이후 OR 저점 이탈 → bearish bias
     - 둘 다 아직 → neutral
  4. 결과를 data/daily/ 폴더에 JSON 저장

출력 형식:
  {
    "date": "2026-03-07",
    "or_high": 72900,
    "or_low": 71800,
    "ir_high": 73200,
    "ir_low": 71500,
    "daily_bias": "bullish",  // or "bearish" or "neutral"
    "bias_trigger_time": "10:14:00",
    "premium_levels": {
      "prev_day_high": 73800,
      "prev_day_low": 71200,
      "prev_week_high": 75400,
      "prev_week_low": 69800
    }
  }

데이터 소스:
  - KIS API 분봉 데이터 (가능 시)
  - 불가 시 1분/5분 캔들 CSV 폴백

연동:
  - 퀀트봇: 스윙 매수 시그널 발생 시 daily_bias와 일치 여부 확인 → 불일치 시 신뢰도 하향
  - 단타봇: OR 브레이크아웃 기준점으로 활용 (Phase 2 모듈 2-2와 연결)
"""
```

- **테스트**: 최근 20거래일 데이터로 OR/IR 기록 → daily_bias와 실제 종가 방향 일치율 확인
- **기대 효과**: daily_bias 일치율 55% 이상이면 유의미 (무작위 50% 대비)

### 모듈 1-3. 프리미엄 유동성 레벨 계산
- **대상**: 전체 봇
- **난이도**: ★★☆
- **목적**: 전일 고/저, 전주 고/저를 "유동성 타겟"으로 자동 계산. 기존 지지/저항 대비 체계적 관리.

```python
# premium_levels.py (신규)

"""
매일 AM 08:30 (정보봇 스케줄) 또는 장 시작 전 자동 계산:

premium_liquidity = {
    "prev_day_high": ...,
    "prev_day_low": ...,
    "prev_week_high": ...,     # 직전 완성된 주봉 기준
    "prev_week_low": ...,
    "prev_month_high": ...,    # 직전 완성된 월봉 기준
    "prev_month_low": ...,
    "or_high": None,           # 장중 09:30에 채워짐 (모듈 1-2)
    "or_low": None,
    "ir_high": None,           # 장중 10:00에 채워짐 (모듈 1-2)
    "ir_low": None,
}

활용:
  - 현재가와 각 레벨의 거리(%) 자동 계산
  - 가장 가까운 프리미엄 레벨을 시그널에 표시
  - 예: "한국콜마 72,400 | 전일고 73,800까지 1.9% | OR저 71,800까지 -0.8%"

저장:
  - data/premium_levels/{date}.json
  - 텔레그램 리포트에 프리미엄 레벨 섹션 추가
"""
```

- **테스트**: 최근 20거래일, 프리미엄 레벨 터치 후 반전 비율 측정
- **기대 효과**: 기존 단순 지지/저항 대비 레벨 체계화 → 타겟/손절 설정 정밀도 향상

## Phase 2 — Phase 1 검증 후 구현 (근거 ★★★, 보완적 가치)

> **착수 조건**: Phase 1 모듈 3개 실전 운용 최소 2주 + OR/IR bias 정확도 55% 이상 확인

### 모듈 2-1. 킬존 시간 필터
- **대상**: 단타봇
- **난이도**: ★☆☆
- **목적**: 점심 세션(11:30~13:00) 신규 진입 자동 차단

```python
# killzone_filter.py (신규)

"""
매매 허용/차단 시간:
  ALLOWED = [
      ("09:30", "11:30"),   # 오전 메인 세션
      ("13:00", "14:30"),   # 오후 세션
  ]
  
  BLOCKED = [
      ("08:00", "09:30"),   # OR 형성 관찰만 (진입 금지)
      ("11:30", "13:00"),   # 점심 세션 (진입 금지)
      ("14:30", "15:30"),   # 마감 세션 (단타 진입 금지)
  ]

동작:
  - 단타봇 주문 함수 앞에 시간 체크 래퍼 추가
  - BLOCKED 시간에 주문 시도 → 차단 + 로그 기록
  - 기존 포지션 관리(손절/익절)는 시간 무관하게 허용

검증:
  - 과거 단타봇 거래 내역에서 점심 세션 진입 건만 추출
  - 해당 건들의 승률/손익 분석 → 전체 평균 대비 열악한지 확인
"""
```

### 모듈 2-2. Equal High/Low 탐지
- **대상**: 퀀트봇 + 정보봇
- **난이도**: ★★★
- **목적**: 동일 가격대에 여러 번 터치된 레벨 = 유동성 축적 포인트 탐지

```python
# equal_level_detector.py (신규)

"""
정의:
  - Equal Low: 최근 N거래일 중 저점이 ±0.3% 이내에 2회 이상 형성된 가격대
  - Equal High: 최근 N거래일 중 고점이 ±0.3% 이내에 2회 이상 형성된 가격대
  - 허용 오차(0.3%)는 호가 단위 감안한 한국 시장 조정값 (백테스트로 최적화)

의미:
  - Equal Low 아래 = 손절 주문 축적 → 이 레벨 이탈 시 "유동성 흡수" 이벤트
  - Equal High 위 = 동일 논리
  - ICT의 "책갈피(bookmark)" 개념 — 시장이 나중에 돌아올 가능성이 높은 레벨

출력:
  {
    "symbol": "한국콜마",
    "equal_lows": [{"price": 68500, "touches": 3, "dates": ["03-01", "03-03", "03-05"]}],
    "equal_highs": [{"price": 75200, "touches": 2, "dates": ["02-28", "03-04"]}]
  }

활용:
  - 정보봇: 스냅샷에 Equal H/L 표시 → "Equal Low 68,500 x3회 (하방 유동성 축적)"
  - 퀀트봇: Equal Low 이탈 후 급반전 = 매수 시그널 신뢰도 상향
"""
```

# risk_cooldown.py (신규 또는 기존 리스크 매니저 확장)

## Phase 3 — 백테스트 결과에 따라 선택적 구현 (근거 ★★☆, 효과 미확인)

> **착수 조건**: Phase 2 완료 + Equal H/L 반전 적중률 데이터 확보

### 모듈 3-1. HP Gap (고확률 갭) 탐지
- **대상**: 퀀트봇
- **난이도**: ★★★
- **근거 주의**: ICT 고유 개념. 학술 검증 없음. 반드시 백테스트 후 채택 여부 결정.

```
정의:
  - 갭(전일 종가 vs 당일 시가 ±0.5% 이상) 발생
  - 그 갭 방향으로 프리미엄 유동성(전일 고/저)을 흡수한 경우 = HP Gap
  - HP Gap은 되돌림 시 강한 지지/저항 역할 (ICT 주장)

구현 전 필수:
  - 최근 6개월 데이터로 백테스트
  - HP Gap 레벨에서 반전 비율 vs 일반 갭 반전 비율 비교
  - 유의미한 차이(>10%) 있을 때만 구현 진행
```

### 모듈 3-2. AMD 패턴 탐지
- **대상**: 단타봇
- **난이도**: ★★★
- **근거 주의**: 사후 해석 편향 위험 높음. 실시간 탐지 정확도가 핵심.

```
정의:
  - Accumulation: OR 구간 레인지 < ATR 50% (좁은 횡보)
  - Manipulation: IR에서 OR 고/저 이탈 + 거래량 감소 (가짜 돌파)
  - Distribution: 이탈 후 반대 방향 급등/급락 + 거래량 급증

구현 전 필수:
  - 최근 3개월 OR/IR 데이터 기반 AMD 패턴 수동 라벨링 (20건 이상)
  - 패턴 발생 후 방향 적중률 측정
  - 60% 이상이면 구현, 미만이면 폐기
```

### 모듈 3-3. ICT 4질문 자동 체크
- **대상**: 정보봇
- **난이도**: ★★★
- **근거 주의**: Phase 1~2 모듈의 종합판. 개별 모듈이 검증되어야 의미 있음.

```
진입 전 4질문 (ICT 원본 → 한국 시장 변환):

  Q1. 프리미엄 유동성 흡수?
      → 전일저/주간저/Equal Low 이탈 후 반등 (Phase 1-3 + Phase 2-2 데이터)
  
  Q2. 급격한 구조 전환?
      → RSI 반전 + 거래량 급증 + 양봉 (기존 v2.1 반전 3조건)
  
  Q3. 균형 가격 형성?
      → 갭 반전 또는 양방향 캔들 겹침 (Phase 3-1 HP Gap 데이터)
  
  Q4. 타겟 유동성 명확?
      → Equal High, 전일고, 프리미엄 레벨 중 타겟 존재 여부

  4개 모두 ✅ → "셋업 완성"
  1개라도 ❌ → "대기"

구현 전 필수:
  - Phase 1, 2의 모든 모듈이 실전 운용 중이어야 함
  - Q1~Q4 각각의 개별 적중률 데이터가 있어야 조합 효과 측정 가능
```

  퀀트봇 (09:00~)

  NIGHTWATCH (PM 17:00)
    ├─ 당일 OR/IR 대비 종가 위치 리뷰
    └─ 익일 프리미엄 레벨 사전 계산

## 한국 시장 적용 시 주의사항

1. **FVG 정의 완화**: 한국 주식은 호가 단위 존재. ICT의 캔들 간 갭 정의를 그대로 쓰면 안 됨.
   → 갭 기준: 시가 vs 전일 종가 ±0.5% 이상으로 설정 (백테스트로 최적화)

2. **타임프레임 차이**: ICT는 1분봉 스캘핑 기반. 퀀트봇은 일봉 스윙.
   → OR/IR 개념은 일봉에서 "당일 첫 1시간"으로 변환. 갭/FVG 개념은 일봉 갭으로 변환.

3. **수급 데이터 결합**: 한국 시장은 개인 비중이 높아 ICT의 "기관이 유동성 사냥" 가정이 약함.
   → 외국인/기관 순매수 데이터와 프리미엄 레벨을 교차 검증해야 신뢰도 상승.

4. **과적합 주의**: ICT 개념을 전부 넣으면 파라미터가 너무 많아짐.
   → Phase별로 1개씩 추가하며 A/B 테스트. 효과 없으면 과감히 제거.
