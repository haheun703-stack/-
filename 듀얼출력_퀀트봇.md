## 핵심 변경사항

지금부터 분석 결과를 **2개 채널로 나눠서 출력**해.
같은 분석을 한 번만 실행하되, 출력만 2갈래로 나뉘는 거야.

### 채널 1: 텔레그램 (개인용 — 사장님 전용)
- **전체 ETF/섹터/그룹 분석** 다 보내
- 모든 ETF 스코어링 결과 (BUY/HOLD/SELL 전부)
- 섹터 로테이션 순위 + "지금 이 섹터 들어가도 되는지" 개인 판단
- 그룹순환 상태 + "현대차그룹 좀 더 기다려야 할 것 같다" 같은 코멘트
- 외국인 자금 흐름 전체 (INFLOW/OUTFLOW/NEUTRAL 전부)
- 페이퍼 트레이딩 진입/청산 시 상세 이유
- 레짐 판단 (MOMENTUM/NORMAL) + 전환 신호 감지
- 리스크 경고, 포지션 조정 제안
- **필터링 없이 날것 그대로**

### 채널 2: Supabase FLOWX (구독자용 — PRO 유료 서비스)
- **필터링된 결과만** 올려
- 깔끔하게 정제된 데이터, 개인 코멘트 제외
- 아래 기준으로 필터링:

**etf_signals 필터 기준:**
- signal이 'BUY' 또는 'SELL'인 것만 (HOLD는 올리지 마)
- score 60점 이상만
- 개인 코멘트/판단 제외, 숫자 데이터만
- sector_rotation_rank는 전부 올려 (순위니까)
- group_tag 있는 ETF는 반드시 포함

**china_flow (외국인 자금 흐름) 필터 기준:**
- signal이 'INFLOW' 또는 'OUTFLOW'인 것만 (NEUTRAL은 올리지 마)
  → 단, 구독자가 "뭐가 움직이는지"만 보면 되니까
  → NEUTRAL이 너무 많으면 노이즈
- z_score 절대값 1.5 이상인 것 우선
- 올릴 때 개인 해석 제외

**paper_trades 필터 기준:**
- 모든 매매 기록은 올려 (투명성이 핵심)
- memo 필드: "외국인 순매수 전환" 같은 객관적 이유만
  → "좀 불안한데 일단 들어감" 같은 개인 감정은 제외
- cumulative 지표 (PF/MDD/승률)는 매번 갱신해서 올려

## 코드 구조 예시

```python
# 분석 실행 (1번만)
etf_results = run_etf_scoring()         # 전체 ETF 스코어링
china_results = run_china_flow()         # 전체 외국인 자금 흐름
trade_result = check_paper_trades()      # 페이퍼 트레이딩 체크

# ===== 채널 1: 텔레그램 (개인용) =====
# ETF 전체 결과
send_telegram_personal(
    text=f"📊 [개인] 퀀트 리포트 {today}\n\n"
         f"🔄 레짐: {current_regime}\n"
         f"💭 {regime_comment}\n\n"                    # 개인 코멘트
         f"📈 ETF BUY:\n{format_etf(etf_buy_list)}\n"
         f"📉 ETF SELL:\n{format_etf(etf_sell_list)}\n"
         f"⏸️ ETF HOLD:\n{format_etf(etf_hold_list)}\n"  # HOLD도 보여줌
         f"\n💭 섹터 코멘트: {sector_comment}\n"        # 개인 판단
         f"\n🌊 외국인 자금:\n"
         f"  INFLOW: {format_china(inflow_list)}\n"
         f"  OUTFLOW: {format_china(outflow_list)}\n"
         f"  NEUTRAL: {len(neutral_list)}개\n"         # NEUTRAL 개수만
         f"\n💭 자금 코멘트: {china_comment}\n"         # 개인 해석
         f"\n⚠️ 리스크: {risk_warning}"
)

# 페이퍼 트레이딩 매매 시
if trade_result:
    send_telegram_personal(
        text=f"{'🟢 BUY' if trade_result['side']=='BUY' else '🔴 SELL'}\n"
             f"{trade_result['name']} @ {trade_result['price']}\n"
             f"전략: {trade_result['strategy']}\n"
             f"이유: {trade_result['detailed_reason']}\n"  # 상세 이유
             f"확신도: {trade_result['confidence']}%\n"     # 확신도
             f"💭 {trade_result['personal_note']}"          # 개인 메모
    )

# ===== 채널 2: Supabase FLOWX (구독자용) =====
# ETF: BUY/SELL만, score 60+ 필터
flowx_etf = [
    {k: v for k, v in etf.items() if k != 'personal_comment'}  # 코멘트 제거
    for etf in etf_results
    if etf['signal'] in ['BUY', 'SELL'] and etf['score'] >= 60
]
upload_etf_signals(flowx_etf)

# 외국인 자금: INFLOW/OUTFLOW만
flowx_china = [
    {k: v for k, v in china.items() if k != 'personal_comment'}
    for china in china_results
    if china['signal'] in ['INFLOW', 'OUTFLOW']
]
upload_china_flow(flowx_china)

# 페이퍼 트레이딩: 매매 기록은 전부 (투명성)
if trade_result:
    upload_paper_trade({
        'trade_date': str(today),
        'code': trade_result['code'],
        'name': trade_result['name'],
        'side': trade_result['side'],
        'price': trade_result['price'],
        'quantity': trade_result['quantity'],
        'pnl_pct': trade_result.get('pnl_pct'),
        'strategy': trade_result['strategy'],
        'cumulative_pf': trade_result['cumulative_pf'],
        'cumulative_mdd': trade_result['cumulative_mdd'],
        'win_rate': trade_result['win_rate'],
        'memo': trade_result['objective_reason']  # 객관적 이유만
        # personal_note, confidence, detailed_reason → FLOWX에 안 올림
    })
```

## 요약

| 채널 | 대상 | 내용 | 필터 |
|------|------|------|------|
| 텔레그램 개인 | 사장님 | ETF 전체 + HOLD 포함 + 코멘트 + 리스크 | 없음 (날것) |
| Supabase FLOWX | PRO 구독자 | BUY/SELL만 + score 60+ + 숫자만 | 엄격 필터 |

---

## 주의사항

- **분석은 1번만 실행.** 텔레그램용 따로, FLOWX용 따로 분석하지 마. 같은 결과를 필터링만 다르게.
- **개인 코멘트 키:** `personal_comment`, `personal_note`, `detailed_reason`, `confidence` 같은 필드를 추가해서 개인용에만 쓰고, Supabase 업로드 시 제외.
- **paper_trades는 전부 올려.** 수익률 투명 공개가 FLOWX의 신뢰 자산이야. 단, memo는 객관적 이유만.
