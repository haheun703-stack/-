## 1. 너의 역할

너(퀀트봇)는 FLOWX 서비스의 **PRO 티어 (₩25,000/월)** 데이터를 공급하는 봇이야.
매일 장 마감 후 ETF 시그널, 섹터 로테이션, 중국자금 흐름을 Supabase DB에 올려야 해.
또한 페이퍼 트레이딩 매매가 발생할 때마다 기록을 올려서 수익률을 투명 공개해야 해.
FLOWX 웹사이트(flowx.kr)가 이 DB에서 데이터를 읽어서 PRO 구독자에게 보여줘.

**너의 담당 패널:**
- ETF 시그널 (BUY/HOLD/SELL + 섹터 로테이션 순위)
- 섹터 모멘텀 (그룹순환: 현대차3+삼성4 등)
- 중국자금 흐름 (Z-score 기반)
- 실전 수익률 (페이퍼 트레이딩 PF/MDD/승률)

**너의 담당 테이블:** `etf_signals`, `china_flow`, `paper_trades` (3개)

## 5. 업로드 코드

아래 코드를 `upload_quant.py`로 저장해.

```python
"""
FLOWX 퀀트봇 → Supabase 업로드 모듈
담당: etf_signals, china_flow, paper_trades
"""
from supabase import create_client
import os
from datetime import date

# Supabase 클라이언트 초기화
supabase = create_client(
    os.environ['SUPABASE_URL'],
    os.environ['SUPABASE_KEY']
)


def upload_etf_signals(rows: list):
    """
    ETF 시그널 업로드 (여러 ETF 한번에).
    
    Parameters:
        rows (list): [
            {
                'date': '2026-03-16',
                'code': '069500',
                'name': 'KODEX 200',
                'signal': 'BUY',
                'score': 82,
                'change_1d': 1.2,
                'change_5d': 3.5,
                'rsi': 65,
                'sector_rotation_rank': 1,
                'group_tag': None  # 그룹 ETF가 아니면 None
            },
            ...
        ]
    """
    try:
        result = supabase.table('etf_signals').upsert(
            rows, on_conflict='date,code'
        ).execute()
        print(f"[FLOWX] ETF 시그널 업로드 완료: {len(rows)}개")
        return result
    except Exception as e:
        print(f"[FLOWX] ETF 시그널 업로드 실패: {e}")
        return None


def upload_china_flow(rows: list):
    """
    중국자금 흐름 업로드.
    
    Parameters:
        rows (list): [
            {
                'date': '2026-03-16',
                'code': '005930',
                'name': '삼성전자',
                'signal': 'INFLOW',
                'score': 78,
                'z_score': 2.3,
                'change_5d_pct': 3.2
            },
            ...
        ]
    """
    try:
        result = supabase.table('china_flow').upsert(
            rows, on_conflict='date,code'
        ).execute()
        print(f"[FLOWX] 중국자금 업로드 완료: {len(rows)}개 종목")
        return result
    except Exception as e:
        print(f"[FLOWX] 중국자금 업로드 실패: {e}")
        return None


def upload_paper_trade(trade: dict):
    """
    페이퍼 트레이딩 매매 기록 업로드.
    매매가 발생할 때마다 호출.
    
    Parameters:
        trade (dict): {
            'trade_date': '2026-03-16',
            'code': '005930',
            'name': '삼성전자',
            'side': 'BUY',           # 'BUY' 또는 'SELL'
            'price': 83500,
            'quantity': 10,
            'pnl_pct': None,         # BUY일 때는 None, SELL일 때 수익률
            'strategy': 'SWING',     # 'SWING' / 'ETF_ROTATION' / 'GROUP_ROTATION'
            'cumulative_pf': 1.78,   # 현재까지 누적 PF
            'cumulative_mdd': -4.5,  # 현재까지 누적 MDD
            'win_rate': 62.5,        # 현재까지 누적 승률
            'memo': '외국인 순매수 전환'
        }
    """
    try:
        result = supabase.table('paper_trades').insert(trade).execute()
        side_emoji = "🟢" if trade['side'] == 'BUY' else "🔴"
        print(f"[FLOWX] {side_emoji} 매매 기록: {trade['side']} {trade['name']} @ {trade['price']}")
        return result
    except Exception as e:
        print(f"[FLOWX] 매매 기록 업로드 실패: {e}")
        return None
```

## 9. 주의사항

- **너는 읽기 하지 마.** 쓰기만 해. 읽기는 웹봇(FLOWX)이 담당.
- **paper_trades는 INSERT만 써.** UPSERT 아님. 같은 종목 여러 번 매매 가능하니까.
- **etf_signals, china_flow는 UPSERT 써.** 같은 날 다시 돌리면 덮어쓰기.
- **cumulative 필드들 (pf, mdd, win_rate):** 매매 기록마다 갱신된 누적 값을 넣어. FLOWX 대시보드에서 최신 row의 cumulative 값을 보여줄 거야.
- **group_tag:** 현대차그룹 ETF면 '현대차그룹', 삼성그룹이면 '삼성그룹', 일반 ETF면 null.

## 10. 섹터맵(sector_universe) 연동 규칙

> 출처: `FLOWX_BOT_INSTRUCTIONS.md` 섹션 4 (2026.03.21)

### 퀀트봇은 자기 테이블에만 INSERT (섹터맵과 직접 무관)

**왜 퀀트봇은 sector_universe를 안 건드리나:**
- 퀀트봇의 데이터(ETF 시그널, 섹터 로테이션, 중국자금)는 **별도 테이블**에 들어감
- 섹터맵(sector_universe)은 정보봇+단타봇이 채우고, 웹봇이 읽음
- 퀀트봇 데이터는 웹봇이 **별도 차트**(PortfolioDonut, ChinaFlowArea 등)에서 읽어감

### 퀀트봇이 쓰는 테이블 (3개만)
| 테이블 | 용도 | FlowX 패널 |
|--------|------|------------|
| `etf_signals` | ETF 매수/매도/홀드 시그널 | ETF 시그널 차트 |
| `china_flow` | 북향/남향 자금 흐름 | 중국자금 흐름 차트 |
| `paper_trades` | 페이퍼 트레이딩 기록 | 누적 수익률 차트 |

### ⚠️ sector_universe 테이블은 읽기만 가능 (CRITICAL)
- 퀀트봇이 5축 스코어로 종목을 평가할 때 sector_universe를 **참조(READ)**는 OK
- 하지만 change_pct나 foreign_net 등을 **수정(UPDATE)**하면 **절대 안 됨**
- 그건 정보봇(change_pct) / 단타봇(foreign_net, institution_net, volume_ratio) 영역

### 향후 연결 가능성
- 퀀트봇의 섹터 로테이션 데이터가 **섹터맵과 연결**될 수 있음
- 예: "지금 자금이 반도체 → 방산으로 이동 중" → 섹터맵에서 화살표로 표시
- 이건 Phase C 이후 논의
